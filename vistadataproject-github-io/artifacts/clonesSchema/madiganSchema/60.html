description
This file holds the specific site information related to each laboratory test or laboratory panel. Entries may be added or modified, but none of the supplied entries should be deleted. The file defines specific information for each test such as synonyms, accession area, collection sample(s), print order, CAP/CPT codes and methods unique to the test. Some pieces of data can be defined for each work element such as test cost, lab print name and order, overdue criteria, and the result optional flag.

3type(+)0;3SET OF CODESB:BOTH
C:CALCULATED
I:INPUT
M:MISCELLANEOUS
N:NEITHER
O:OUTPUT
B - (Both) means test can be both ordered and resulted/reported. C - (Calculated) means test result is automatically calculated by the system. I - (Input) means test can be ordered, but not resulted/reported by itself. M - (Miscellaneous) specifies test that can be re-defined for local use. N - (Neither) means test can no longer be ordered and/or resulted. O - (Output) means test can be resulted/reported, but not ordered. The entry in this field determines how the system handles a test for ordering, resulting, and reporting. Glucose, TSH, Retic, Occult Blood, etc. are examples of tests that normally have the "Type" field set to BOTH (meaning these tests can be both ordered and resulted/reported individually). This represents the majority of tests in the system. An example of a CALCULATED test is Calc. LDL. Results for the Calc. LDL test are automatically calculated by the system from the results of the TRIGLYCERIDES, CHOLESTEROL, and HDL tests. An example of an INPUT test is a CBC. An order is placed for a CBC, but results are entered and reported for the individual tests of the CBC (i.e. WBC, RBC, HGB, etc). Single tests do not usually have the "Type" field set to INPUT. Tests designated as MISCELLANEOUS are used to allow the site to set up new tests as necessary. Tests already assigned as another type cannot be changed to miscellaneous. Miscellaneous tests are pre-defined in the system and cannot be added by the site. Whenever a site re-defines a miscellaneous test, a mail bulletin is generated to a group authorized to evaluate the test for inclusion as a DOD Standard Lab Test. Tests can be made unorderable by changing the "Type" field to NEITHER. If there is no need to order, edit, or display a test, it may be inactivated by setting the "Inactive Flag" field to YES. Once a test has been made inactive, it cannot be re-activated. Deleting entries from the Lab Test file is not allowed, since database integrity will be compromised. Components of a panel are examples of tests that normally have the "Type" field set to OUTPUT. Specific examples are tests such as Appearance, Turbidity, Urine Glucose, Crystals, etc. that are always ordered as part of a Urinalysis panel.

Enter a subscript for the Lab Test entry. Examples are CH for Clinical Chemistry, BT for Bacteriology, and SP for Surgical Pathology. Each subscript dictates a unique set of order and result entry requirements for various lab tests. The Clinical Chemistry subscript allows the user to enter a single result for each test. Use CH for most laboratory tests including Chemistry, Hematology, Immunology, and Coagulation tests. New tests with a CH subscript can be added to the system only as panel tests or QC ONLY tests. The subscripts for Bacteriology, Parasitology, Mycobacterium, Mycology, and Virology comprise the Microbiology test group. Each subscript provides the user with a specialized result entry screen which is determined by the Micro Screen field designated in the Lab Test file. New tests with a Microbiology subscript can be added to the system only as panel tests or QC ONLY tests. The subscripts for Autopsy, Bone Marrow, Cytology Gynecological, Cytology Non-Gynecological, and Surgical Pathology comprise the Anatomic Pathology test group. Each subscript provides the user with a specialized result entry screen. New tests cannot be added with an Anatomic Pathology subscript. A standard set of tests with the Blood Bank (BB) subscript have been added to the Lab Test file as part of the interface between CHCS and DBSS (Defense Blood Standard System). New blood bank tests may not be added to the Lab Test file.

1:YES
Enter YES if this test is for QC purposes only, NO if test is a panel, or "^" to exit. If a single test, enter "YES" to designate test as QC ONLY. Single tests that are not used strictly for quality control purposes cannot be added to this file. If a panel test, enter '^' to delete test or 'NO' to add panel test as a non-QC test. Tests that are QC ONLY will usually be done in the Microbiology areas (Bacteriology, Parasitology, Mycology, Mycobacteriology, Virology) and Clinical Chemistry. 703duplicate check time0;18NUMERICThis is the number of hours or days (1-60) for duplicate check enforcement used in conjunction with the duplicate check interval. Each time the test is ordered the system checks for potential duplicates within the time frame indicated. If it is 1 hour, the system would only check for a duplicate order for the hour prior and an hour in the future. All other orders would be ignored. The duplicate check is only a warning, and this warning is easily overridden by the HCP. Any duplicates ordered during order entry will further be checked by the system when the collection list is printed and log-in occurs. Any duplicates are cancelled at that time by the system as duplicate orders. If this field is blank the duplicate check will not occur during order entry.
