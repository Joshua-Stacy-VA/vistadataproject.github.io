Default = OFF The intent of this field is for debugging purposes. If the mode is turned on, the raw transmission data that is returned from DEERS is stored in the appropriate response file. Running CHCS DEERS in the debug mode may have an adverse effect on performance.
.08training environment0;8BOOLEAN0:NO
1:YES
Default = NO If this field is set to "yes", the CHCS DEERS software will not send DEERS requests. It will only simulate the process for training classes. This field must be set to "no" in a production environment.
.09transaction statistics0;9SET OF CODES0:MONTHLY
1:WEEKLY
2:DAILY
Default = MONTHLY The number of transactions sent to DEERS is recorded in the DEERS Transmission Statistics file monthly by default. No entry is required. If the information is needed on a more frequent basis, the collection frequency can be set to weekly or daily.
.1outpatient visit0;10NUMERICThis field is used by the PAS Pre-booked appointment functionality to determine how many non-interactive requests the PAS functionality should post to the transmission queue. CHCS requirements state that PAS must perform eligibility checks on no less than 25% of outpatient visits. The CHCS DEERS software does not use this parameter.
.11civ prescription0;11NUMERICDefault = 0 This field exists to satisfy a requirement for system manager capability to modify DEERS query percentages. However, the PHR software does not use this parameter since 100% of civilian prescriptions are required to have an eligibility check. The CHCS DEERS software does not use this field. There is no need to enter a value in this field.
.12admission0;12NUMERICDefault = 0 This field exists to satisfy a requirement for system manager capability to modify DEERS query percentages. However, the PAD software does not use this parameter since 100% of inpatient admissions are required to have an eligibility check. The CHCS DEERS software does not use this field. There is no need to enter a value in this field.
.13external response time0;13NUMERICDefault = 8 External DEERS is obligated to return a response to CHCS within 7 seconds. If the external DEERS processing time equals or exceeds the amount entered in this field, the time exceeded will be written to the DEERS Error Log file. If a 0 is entered, this functionality is turned off.
.14maximum error entries0;14NUMERICWhen the DEERS error log is purged by the nightly task, it will keep at most this many entries in the file.
