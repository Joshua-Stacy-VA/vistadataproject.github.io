.01site/specimen(+)0;1POINTER61BThis is the site/specimen for the lab test for the cumulative. If this test is done on several site/specimens (e.g. fluid tests) then a separate entry is made for each specimen and the format is defined for each specimen.
.02display name0;2FREE TEXTThis is the name of the test as it displays on the cumulative report. You may use a name different than the print names defined in the LAB TEST file, since a combination of the header name and print name may be a better way to display the information. (e.g. header of "Glucose Tolerance" and test display name of "FASTING, 1/2HR., 1HR. etc.)
.03header(+)0;3POINTER65.1This is the name of the header where the test for this site/specimen shall reside. The header must first be defined in the LAB CUMULATIVE HEADER file. The lab may want to include in the header that the cumulative replaces previous forms.
.04field length(+)0;4NUMERICThis is the length of the field allowed for this result. The length of this field will determine how many tests or results can appear in a format.
.05display order(+)0;5NUMERICThis is the order of display of the tests for this site/specimen and header. The tests for a header must be the same site/specimen but do not necessarily have the same accession area. Therefore, the display order of the tests may be different from those is PLI. Although the lab should try to have the tests display in somewhat the same order as PLI. This will create a sense of consistency as the HCP moves from one reporting option to another.
