description
This file contains all orders, both inpatient and outpatient for all patients in the system. The fields are grouped as follows: Subscript 0 - all fields whose primary purpose is to uniquely identify an order and link it to the patient, encounter and episode. Subscript 1 - the fields responsible for tracking order signatures and authorizations. Subscript 2 - all order status flags and status tracking fields. Includes order priority flags (Eg.STAT). Subscript 3 - order modification fields Subscript 5 - multiple sequential orders Subscript 50 - subfile for tracking secondary orders Subscript 51 - subfile to track Warnings issued Subscript 52 - multiple subfile for requisitioning and resulting orders Subscript 53 - subfile for tracking changes in order status Subscript 101 - multiple subfile for tracking QA 'failed events' Subscripts 110-114 - Laboratory orders Subscripts 115-119 - Nursing/Intraward orders Subscripts 120-129 - Pharmacy Subscripts 125-129 - Diet orders Subscripts 130-134 - Radiology orders Subscripts 150-154 - Ancillary orders Subscripts 160-164 - ADT orders

2.03inactive status2;3SET OF CODES0:RESULTED

2:TRANSFERRED
3:SIGNATURE MISSING
4:RESULTED-NO SIG
This flag further defines the status of an inactive order. If RESULTED, the order has had all tasks resulted. EXPLICITLY DISCONTINUED means that an action was taken by a provider to stop this order. TRANSFERRED is the inactive status created when a patient is transferred within the MTF. At that point, all orders are 'discontinued' but the provider has an opportunity to later mark which of these orders he would like to 'reactivate'. This reactivation actually results in new orders being generated. SIGNATURE MISSING means that an order has been discontinued but still needs a signature or countersignature. It will continue to show up on the display screen until signed. If a provider cancels an order requiring a signature, and he is qualified to sign that order, the signing takes place automatically and this status is avoided.
