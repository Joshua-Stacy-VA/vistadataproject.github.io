.01field number within sector(+)0;1NUMERIC This is the number of the current field for which you are defining type, content, dimensions, and attributes. It is merely a sequence number and has no bearing on how or where the field prints on the label. It is necessary so that each entry may be unique per sector. If, for instance, the user wishes to print the Accession Number as both human-readable text and as a bar code within the same sector, there must be a way to set apart one from another. If not, when the user first defined the field as a bar code, and then tried to add it again to the same sector as text, the user would be thrown back into editing the previous entry (the bar coded field), rather than being allowed to enter the same field again as text.
.03display name(+)0;3FREE TEXT This is the display name for the element you wish to define on the label. It must be a valid entry in the LABEL FIELDS file, and will be displayed on the 'pick list'.
.04user-definable text(+)0;4FREE TEXT This is the text for this field, exactly as it should be printed. It will appear exactly this way and in this location on EVERY label that uses this label design.
.05horizontal starting point(+)0;5NUMERIC This is the horizontal, or 'X', coordinate of the upper left corner of the field. The value must range between zero and the sector width, as specified in the Label Description File. Be careful not to place a field within the space of another field. Defining adjacent fields requires at least one millionth of an inch (.000001") of space between each field.
.06vertical starting point(+)0;6NUMERIC This is the vertical, or 'Y', coordinate of the upper left corner of the field. The value must range between zero and the sector height, as specified in the Label Description File. Be careful not to place a field within the space of another field. Defining adjacent fields requires at least one millionth of an inch (.000001") of space between each field.
.07field width(+)0;7NUMERIC Enter the width, in inches, of the field being defined. The field width must be less than or equal to the distance between the horizontal starting point and the sector width, as specified in the Label Description File. This value specifies the size of the field from left to right, and is used to figure out if the field spatially overlaps other already defined fields.
.08field height(+)0;8NUMERIC This is the height, in inches, of the field being defined. The value must be less than or equal to the distance between the vertical starting point and the sector height, as specified in the Label Description File. This value specifies the size of the field from top to bottom, and is used to figure out if the field spatially overlaps other already defined fields.
