description
The CPT Modifier Calculations File contains business rule processing logic that determines whether a specific CPT modifier impacts the calculation of a rate for a rendered service under OIB.
Fields

.01cpt modifier(+)0;1FREE TEXTBThe CPT modifier is captured in the CHCS Ancillary and Ambulatory Care modules for specific CPT codes. If there is a modifier associated with a CPT code, it may change the display units, act as a multiplier, or drive the type of rate to apply to the CPT code itself. This file stores the factors associated with valid CPT modifiers.
1display units(+)0;2NUMERICThis field indicates the number of units that will display when this modifier is used for a CPT code.
2mod1 uos indicator0;3NUMERICFor some modifiers, Units of Service are used as the quantity for calculating charges for the first modifier. If this is the case, this field will be set to 1. If a different multiplier is used for the first modifier calculation, this field will be set to 0, and the "MOD 1" modifier will be equal to the quantity that will override the units of service on file for the line item.
3display modifier0;4FREE TEXTIn certain cases, the modifier entered will not be displayed, or a different display will be used. For example, a -32 modifier will display as -TC to indicate a technical component for radiology, and -00 will display null.
4secondary multiplier0;5NUMERICSome of the modifiers will only be calculated if they appear as the first modifier for a CPT code. If so, this field will be set to 0 to indicate that no charges are calculated for the 2nd or 3rd modifiers. If billing is allowed for the 2nd or 3rd modifier, this field will be set to 0 if the secondary calculations uses units of service, or will be set to the modifier to be used for calculations.

Otherwise, it will be null or 0. If this field is null, and services are billable, the multiplier value will be found in the secondary multiplier field.
