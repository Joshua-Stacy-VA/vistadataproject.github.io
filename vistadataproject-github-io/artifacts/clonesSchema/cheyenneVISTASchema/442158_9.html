This field is available to the ICR Application Coordinator to be able to provide a pointer to the patient's identity within the PATIENT file. This is primarily designed for those sites which have been using a prior version of ICR software, and may have templates and/or routines which prior versions were pointers to the patient file. If the 'YES' response is selected, the 'NAME' field will be set to the pointer to the patient file for each patient. .11no new patient after message #0;11NUMERICThis variable is used to set a general limit to the number of messages which can be sent by the program collecting data for the National Registry database. The value can be a number of 50 or greater, and if no value is entered, the default value of 75 will be used. Once a number is patient will not be started if the specified number of messages have already been sent. The number of messages generated and sent can exceed the specified number if a patient is started prior to this number and a number of messages sent for the current patient. This permits the site to insure that the number of messages generated for transmission do not overwhelm the electronic mail system.
