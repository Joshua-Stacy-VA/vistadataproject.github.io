.02ip/op indicator0;2SET OF CODES0:BOTH

Indicates if this drug may be used for inpatients, outpatients or both.
.03local cost0;3NUMERICThe LOCAL COST represents the cost of the basic unit of quantity in which a drug is dispensed. The LOCAL COST should correspond to either the METRIC UNITS or the PACKAGE SIZE. For example, if a drug is dispensed in any number of MLs or tablets ("Eaches"), the LOCAL COST should reflect the cost of 1 ML or 1 TAB ("Each") respectively. If a drug has a PACKAGE SIZE defined and is normally dispensed as a 120 ML bottle or 100 TAB bottle (e.g., Nitroglycerin), the LOCAL COST should reflect the cost of 1 package.
.04pdts unit cost0;4NUMERICThis is the unit cost of the drug calculated from the fill cost returned by PDTS (Pharmacy Data Transaction Service). NOTE: For non-compounded (single NDC) items, if the package size or the NDC Number is changed or deleted, that item's PDTS Unit Cost will be deleted across all formulary groups. Historical PDTS Fill Costs will not be affected.
.05non-formulary0;5SET OF CODES1:N/F
0:FORMULARY
This field determines whether the drug is FORMULARY or NON-FORMULARY for this Formulary Group. If a drug is FORMULARY for at least one Formulary Group in the MTF, the drug can be ordered throughout the MTF. If a drug is NON-FORMULARY at all Formulary Groups in the MTF, the drug cannot be ordered.
.06message0;6FREE TEXTComments that are entered here will be displayed along with the drug on a drug look-up. This is a good place to put short messages to the HCPs regarding the dispensing of this medication.
.07local/pdts cost switch0;7SET OF CODESL:LOCAL
P:PDTS
This field specifies whether PDTS (P) or Local Cost (L) is to be displayed on drug lookups and used to calculate the fill cost of a prescription for this drug. PDTS means that data supplied by the Pharmacy Data Transaction Service after the first prescription is entered for an item will be used to calculate the fill cost. LOCAL means that the cost entered in the Local Cost field will be used to calculate the fill cost. NOTE: For non-compounded (single NDC) items, if the package size or the NDC Number is changed or deleted, that item's PDTS Unit Cost will be deleted across all formulary groups. Historical PDTS Fill Costs will not be affected.

1.02maximum days supply1;2NUMERICIf there is a maximum days supply allowed to be prescribed for this drug, enter it here. 1.03maximum refills allowed1;3NUMERICIf there is a maximum refills allowed to be prescribed for this drug, enter it here. If the number of refills entered during new prescription entry by an HCP exceeds the maximum refills, the system will inform the HCP and require that an allowable number of refills be entered. During new prescription entry by Pharmacy, the system will inform the user if the number of refills exceeds the maximum but will not require compliance with the maximum limit.
1.04dispense complete container1;4SET OF CODES1:YES

1.05warning label1;5FREE TEXTEnter the number(s) of the auxiliary warning labels associated with this drug. Separate the numbers entered by commas, ex. 5,3,12. The numbers correspond to the numbers of your auxiliary label dispenser and will be printed on the pharmacy retained portion of the RX label. This will alert the person filling the prescription to attach the appropriate auxiliary warning label to the prescription bottle. Enter the number(s) separated by commas. For example, '2,13,5'.
1.06default expiration1;6NUMERICEnter the number of days that a prescription for this drug is valid according to MTF or DEA policy. For example, Schedule 2 narcotics expire after 3 days. Most drugs will be 6 months (180 days) or 1 year (365 days).
1.07default quantity1;7NUMERICIf there is a commonly prescribed quantity for this drug (e.g. 30 day supply with default sig) enter it here. This quantity will be the default when an OUTPATIENT prescription for this drug is entered.
1.08default sig1;8FREE TEXTIf there is a standard Sig for this medication, enter it here. This Sig will be displayed as a default in the Sig field whenever this drug is ordered on an OUTPATIENT prescription.
1.09default days supply1;9NUMERICThis field specifies the Days Supply for a drug. An entry should only be made when the system cannot calculate the correct Days Supply, as system calculations will not be used.
