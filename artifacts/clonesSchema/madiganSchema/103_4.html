description
This file stores a list of CHCS filenames that are eligible for the Order Required Data functionality. User can set up Order Required Data questions and prompts that specify the responses are stored in a CHCS file. Those CHCS files must be pre-defined in this file. Entries in this file are maintained by the CHCS Development organization and should never be manipulated by site personnel.

.01file name(+)0;1FREE TEXTBThis is the file name from the Fileman Data Dictionaries in CHCS that can be used to supply responses to Order Required Data questions during order entry. The file name entered here must be defined as a valid CHCS file, as listed in the File of Files (file #2). Entries in this file are controlled by the software development organization and should never be manipulated by the sites.
.02source data provided by0;2SET OF CODES1:CHCS I
2:CHCS 2
This field specifies which system will be the source of the data when a question's response type specifies this file. The source of the data can be either CHCS II if the CHCS II application database is maintained in sychronization with the corresponding file on CHCS I, or the source could be CHCS I, in which case data will always be pulled from CHCS I.
.03alternate display field0;3NUMERICThis field specifies an additional display field that will be used during lookup and selection activities. The .01 field of a file will always be the primary display field, and it this field is specified, it will also display to the user. An example of when to specify an additional display field, is when using a file whose .01 field is a coded value of limited length, such as the Lab Specimen Source Code file (8732). This file has a code value as the .01 field, making a lookup and selection list somewhat vague and ambiguous. Populating this field allows for more information about each entry to be displayed to help the user make the appropriate selection.
