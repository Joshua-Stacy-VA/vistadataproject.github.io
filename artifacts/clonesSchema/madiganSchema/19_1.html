description
Each entry in this File is the name of a "key" that "unlocks" some capability for individual specified users. The capability might be access to some OPTION (File 19) which is "locked" with the key's name. Other functionalities (such as allowing a scheduling overbook) might also be "locked" with these security keys. Keys are named with the standard package prefix formats; a 2-4 character package prefix for standard names, or beginning with the letter "A" or "Z".
