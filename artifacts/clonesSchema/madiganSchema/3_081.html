This is the reason the user was denied access. If correct access code is entered with an invalid verify code, the user can be identified and Reason will be set to 1 (Invalid Access Attempt). If the user is unsuccessful after n attempts, Reason will be set to 2 (Locked - Bad Logon), and terminal access will be locked n seconds ("n" is defined in the Kernel Site Parameter file). If an incorrect access code is entered, the user will be identifiable only through $I and CPU signed on to (DUZ=0).
