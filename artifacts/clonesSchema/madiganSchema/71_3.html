description
The Procedure Class file contains the names of the procedure classes (e.g., ULTRASOUND, CHEST, CT) under which similar radiology procedures can be grouped.
Fields
#NameLocationTypeDetailsIndexDescription
.01name(+)0;1FREE TEXTBEnter a 3- to 30-character name of the procedure class (for example, ULTRASOUND, CT, or MRI). Procedure classes are used as a means of grouping similar procedures for the purpose of assigning them duplicate checking, order approval, and report verification parameters. NOTE: All procedures must be assigned to a procedure class.
.02supervising radiologist0;2POINTER6Enter the name of a staff radiologist who is qualified to verify reports for all procedures assigned to this procedure class. This field is used in conjunction with the second signature functionality. If the following conditions have been met: 1) the radiology reports reach a status of PRELIMINARY (that is, they have been interpreted by a resident), and 2) they have been in the supervising radiologist's queue for a site-defined number of days without being verified, the CHCS pulls these reports from the supervising radiologist's queue and places them in the queue of the staff radiologist identified in this field.
.03type of class(+)0;3FREE TEXTThis field defines the duplicate checking, order approval, and verification parameters for procedures that fall within this procedure class. Enter any one of the following codes: D for duplicate checking only O for order approval only R for report verification only DO for duplicate checking and order approval DR for duplicate checking and report verification OR for order approval and report verification DOR for duplicate checking, order approval, and report verification. Set this field to D and the CHCS will perform duplicate checking on all procedures within this procedure class. If a procedure is assigned to a procedure class on which duplicate checking is performed, then during order entry the CHCS checks the current patient's Exam file entries to see if a radiology exam in this procedure class has been ordered for this patient within the site-specified number of days. If a duplicate is found the CHCS displays a warning message stating such. This warning message can be overridden. Set this field to O and the CHCS will require order approval for procedures in this class. When users order procedures in this class, the system displays a message stating that order approval is required. The CHCS will allow radiologists who have this procedure class in their ORDERING APPROVING CLASSES field to approve orders for procedures in this class. Set this field to R and the CHCS will allow verification of reports by radiologists authorized to approve reports for procedures within this class. The CHCS will allow radiologists who have this class in their REPORT APPROVING CLASSES field to verify reports for procedure within this class. You may also combine any of the three settings so that one, two, or all three of these procedure class types may be assigned to a procedure class.
.04imaging type(+)0;4POINTER79.2This field contains the name of this type of imaging (for example, RADIOLOGY).
.11days to check for duplicates0;11NUMERIC DUPLICATES Enter the number of days (from 0 to 365) you want the CHCS to check the patient's Exam file entries for duplicate ordered and/or performed radiology procedures. The CHCS checks for duplicate procedures for the number of days prior to, on, and after the date the newly ordered procedure is scheduled to be performed.
71procedure71;0MULTIPLE71.31Enter the name(s) of the procedure(s) to be included in this procedure class. Assigning a procedure to a procedure class enables the CHCS, when the procedure is ordered, to perform the checks as defined in the TYPE OF CLASS field.
