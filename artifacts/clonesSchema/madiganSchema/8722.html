BAR CODE FORMAT

8722

^LAB(8722,
description
This file contains the physical characteristics of data encoded into a bar code that is required by a given scanner. It includes the symbology, bar width and ratio, quiet zone, check digit, and other specific requirements.
Fields
#NameLocationTypeDetailsIndexDescription
.01bar code format name(+)0;1FREE TEXTBThis is the name of the individual bar code format that will be printed.
.02bar code width(+)0;2NUMERICThis is the maximum width, in inches, of the bar code. It includes the beginning and ending 'Quiet Zones'. The width is calculated in relationship to printing the bar code in the picket fence style of orientation, which has the bar code pattern scannable from left-to-right when printed.
.03bar code height(+)0;3NUMERICThis is the minimum length of each individual bar in this bar code. The height is calculated in relationship to printing the bar code in the picket fence style of orientation, which has the bar code pattern scannble from left-to-right when printed.
.04narrow bar width(+)0;4NUMERICThis is the width, in inches, of the narrowest bar in the bar code which you are defining. When printing the bar code, the value specified here or the printer's minimum resolution, whichever is greater, will be used.
.05bar ratio(+)0;5NUMERICThis is the ratio of the wide bar width to the narrow bar width for the bar code you are defining.
.06minimum quiet zone(+)0;6NUMERICThis is the minimum length, in inches, of the white, or 'quiet' space that the scanner requires before and after the bar code. This informs the scanner that a bar code is beginning and ending, and determines the direction of the scan operation.
.08minimum # of characters(+)0;8NUMERICThis is the minimum number of characters that can be encoded with using this bar code format. When printing data with this bar code format if the actual application data is less than this value, the data will be padded on the left with the value specified for the Leading Character. This value does not include the Start/Stop patterns.
.09maximum # of characters(+)0;9NUMERICThis is the maximum number of characters that can be encoded using this bar code format. This value does not include the Start/Stop patterns or the check digit. The data being encoded could be affected by the FORMATTING LOGIC field, therefore, the maximum length will be checked after the FORMATTING LOGIC has been completed.
.1symbology(+)0;10POINTER8910This is the symbology, or bar code language in which the data will be encoded. It must be an entry in the Bar Code Character Set (#8910) file.
.11start pattern0;11FREE TEXTThis is the pattern that indicates to the scanner that a bar code is beginning. It must be a combination of bars and spaces. Use the following code letters for bars and spaces: N for Narrow Bars, n for Narrow Space, W for Wide Bar, and w for Wide Space.
.12stop pattern0;12FREE TEXTThis is the pattern that indicates to the scanner that a bar code is ending. It must be a combination of bars and spaces. Use the following code letters for bars and spaces: N for Narrow Bars, n for Narrow Space, W for Wide Bar, and w for Wide Space.
.13leading character0;13FREE TEXTThis is the character that may be used, if required, to pad the data that is to be encoded in the bar code.
.14check digit0;14POINTER8911This is the check digit algorithm used to ensure the integrity and accuracy of the scanned bar code.
.15formatting logic1;E1,245MUMPSThis is the executable MUMPS code that may be used, if required, to reformat the data to be encoded, prior to the actual encoding process. The data being encoded is contained in the variable XBARTXT. The MUMPS code here must reset XBARTXT to the value that is desired to be encoded. Any routines that are referenced in this code must be in the application's namespace.
