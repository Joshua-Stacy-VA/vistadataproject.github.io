description
This file contains all the user-specific data related to the Desktop, and other Clinical Interface functions. The entries point to the User file, and this file may be thought of as an extension of it. There are numerous fields that deal with user preferences. There are also 4 subfiles, containing multiple entries for View Text, and Patient Lists, Patient Text & Topics, and Macro Commands. There are multiple cross-references which contain dynamic data related to Desktop functions, such as the last Desktop list, what user is currently in the Desktop, what user has locked their desktop, etc. Each user accessible field has help text associated with it, that can be reviewed from within the Desktop via the Help action.

2:WARD
3:IN-PT
4:OUT-PT
9:CURRENT ID
This field determines the initial patient Display on entry into the Desktop. If it is not set or set to "ALL", then all the patients will be displayed in alphabetical order by patient name. If it is set to "WARD", then all the patients will be displayed sorted by Room-Bed, grouped by ward. If it is set to "IN-PT", then all of the in-patients will be displayed alphabetically by name. If it is set to "OUT-PT", then all of the out-patients will be displayed alphabetically by name. If it is set to "CURRENT ID", then all of the patients will be displayed sorted first by the current identifier. If any subgrouping is selected, and there are no patients in it, then the display will revert to "ALL".
1.05list save1;5SET OF CODES0:ASK
1:MERGE
-1:REPLACE
This field determines what happens if the user saves a list of items to a permanent user file which already exists. If it is not set or set to "ASK", then the user will be prompted whether to replace the existing file with the new items, or merge the newest items with the existing ones. If it is set to "MERGE", then the user will not be prompted, but the new items will automatically be merged with the existing ones. If it is set to "REPLACE", then the user will not be prompted, but the old items will be deleted, and the new items stored under the chosen List name.

If it is set to a positive number, then it will wait that many seconds, before continuing. The main purpose of this field is to allow you to set your Desktop to restore, and still let you abort the restore procedure under specific circumstances, without having to change your preferences.

10.01queued view text10;1SET OF CODES0:LEAVE EXISTING VIEW TEXT ALONE
1:OVERWRITE EXISTING VIEW TEXT
This field determines whether a report Queued to the Desktop View Text subfiles can overwrite an existing entry. When the user is at a DEVICE prompt and selects CHDEVICE (synonym: DESKVIEW), the user may be prompted for a name if the application is interactive, or it will be named automatically when queued. If the site has set a limit on the number of View Texts the user can have, and the user has reached the limit, then an interactive application will inform the user of this, and ask the user what to name the entry it is about to overwrite; the user may abort by entering a '^'. If the application is not interactive, but queued then the system will either automatically overwrite an existing entry, or quit the report. If this is not set or set to "LEAVE"... then a queued report will not overwrite any existing entries. If this is set to "OVERWRITE...", then a queued report will overwrite the first (oldest) View Text entry that exists, if it needs to.

22patient list22;0MULTIPLE8820.022This subfile contains the Patient List entries. Each entry has a Name, a Creation date, an Access field, a word processing Description, and multiple Patient entries. The Patient Lists may be created interactively, or automatically by the Patient List Addend Desk Action. To create a new list interactively, the user should use PLE to get to the screen showing the current (if any) Patient Lists. Place the cursor below the last list entry, and type the name of the new list. The system will ask if the user is adding a new list, and after confirmation, present the next screen for completion. To add a new patient to the list, place the cursor below the last patient, and type the name of the desired patient. The system will then add the patient to the list, and present the next screen for completion. To create a list automatically, user the PL+ to save the selected Desktop patients to this subfile. After the patients are chosen, the system will prompt the user for a Patient List name. If a new name is chosen, the system will save these patients under that name after confirmation that the user is creating a new list. If the user chooses an existing name, the system will either ask, merge, or replace the existing list, based on the user's preferences.
25patient text25;0MULTIPLE8820.025This subfile contains the Patient Text entries. Each entry has a patient, an Entry date, and multiple Topics. The Patient Texts may be created interactively, automatically by pressing the F9 key when browsing the patients in the Desktop, by the PTSE (Patient Text Selected Edit) Desk Action, or the PLTE Patient List to Text Edit) Desk Action. To create a new entry interactively, the user should use PTE to get to the screen showing the current (if any) Patient Texts. Place the cursor below the last patient entry, and type the name of the new patient. The system will look up the patient, then ask if you are adding a new entry. Upon confirmation, you will be presented with the next screen, which will allow entry/editing of Topics. To add a new Topic to the Patient Text, place the cursor below the last Topic, and type the name of the new Topic. Upon confirmation, the system will then add the topic to the list, and present the next screen for completion. To add a patient automatically, browse the patients in your Desktop, and when the Cursor is to the left of the desired Patient, press the F9 key. You will then be in the screen for adding/editing Topics. To add/edit Topics on multiple patients, use the PTS action. After the Desktop patients are selected, the system will cycle through the list asking for confirmation after the 2nd patient. Upon confirmation you will be presented with the screen for adding/editing Topics.
