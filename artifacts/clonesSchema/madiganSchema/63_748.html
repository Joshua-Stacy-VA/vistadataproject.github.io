.01parasite(+)0;1POINTER61.2BEnter the name of the parasite identified. This field points to the Etiology Field file and has a screen so that the user can only select parasites for a result. If a parasite is not available, have the lab manager check the Etiology Field file for the entry. If the entry exists in the file, the identifier must be a parasite. If the entry is not included in the file, the lab manager may add it. Make sure the identifier is a parasite.
.02parasite loinc code0;2FREE TEXTThis field is the LOINC Code assigned by the system for this test result. If the system can determine the LOINC code, it is automatically assigned. For Parasitology, a LOINC code is captured for each parasite identified. Data in this field is the actual LOINC code value determined for this test. The data is the same as the .01 field of the LOINC Codes file (file #8199). Note this field is NOT a pointer field. This is to allow test results and LOINC codes to be assigned that may not be in the LOINC codes file. A LOINC code which is not in CHCS may be captured from an off-board system via an HL-7 message when the off-board system is using a different version of the LOINC tables. Data in this field should not be modified by the user and is generally hidden from most data entry mechanisms.
.03quantity0;3FREE TEXTThis field is used to quantify the amount of organism isolated. Examples: 1+, 2+, 3+ or >100,000 colonies/ul.
