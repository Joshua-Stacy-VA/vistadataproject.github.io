.11main tcp port0;2NUMERICThis is the TCP port number to be used by the master process for this realm. HTTP servers use port 80 by default, but any unassigned port number may be used. Check with your network administrator for guidance. All HTTP clients will attach to this port for their initial connection. If secondary ports are specified, the client will immediately be redirected to a free secondary port, if one is available. If no secondary ports are specified, all communication between the client and server is done through the main port. No other client will be able to connect while an active connection is in progress. Under VMS, if this realm is being serviced by UCX then no port number is required. See the WebMan documentation for information on setting up WebMan under UCX control.

.5home directory.5;E1,240FREE TEXTThis defines the physical directory (path) which will serve as the logical root directory for the realm. It should be specified in the format appropriate for the underlying operating system. For example, if the operating system is OpenVMS and the value of this field is DISK1:[WEBMAN], then all URLs passed to this realm will be relative to this directory. If the URL is: http://indianapolis.med.va.gov/ftp/file.zip WebMan would look for the file in the physical directory DISK1:[WEBMAN.FTP] This insures that clients can only access files within and descendant from the specified directory. Of course, operating system file protections ultimately determine whether file access is permitted. It is therefore important to insure that the WebMan background processes have been granted access to the necessary directories and files.

Otherwise, this field should contain code that sets the value of $T to true if the user has been authenticated, or false if not.
