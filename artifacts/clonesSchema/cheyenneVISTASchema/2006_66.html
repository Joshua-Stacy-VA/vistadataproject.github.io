1auto-display?0;2BOOLEAN1:YES
0:NO
If prior cases having this CPT GROUP should be automatically displayed when reading a Current Case, then the answer should be 'YES'; if not, the answer should be 'NO'. By allowing this YES/NO for each entry defined, it becomes possible to define rules flexibly, using combinations of Inclusive and Exclusive logic. For example, a possible definition for the CURRENT CASE CPT GROUP for 735: CURRENT PRIOR AUTO-DISPLAY? 735 735 Y 7356 N 7358 N 7359 N This example says, for a Current Case with CPT code beginning with 735, do the Auto-Display for any Prior Case having a CPT of 735nn, except for those having CPTs of 7356n, 7358n, and 7359n. A more complete definition of this example would then (likely) define three CURRENT entries for these latter 4-digit groups.
