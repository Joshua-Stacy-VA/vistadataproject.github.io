description
This file contains information pertaining to all Standard Inpatient Data Record (SIDR) edits. These edits are performed against various inpatient data stored in the PATIENT (#2) and ENCOUNTER (#311) files. The contents of this file are standar d for all of CHCS and must not be modified.

.01edit number(+)0;1FREE TEXTBThe number of the edit as specified in Appendix B of the Standard Inpatient Data Record Functional Description and in the Fiscal Year SIDR updates. If the edit can be divided into separate sub-edits, number each sub-edit sequentially as a decimal, beginning with 1 (i.e., 105.1). If the edit/sub-edit has effective dates, concatenate an alphabetic character to the end of each edit/sub-edit, beginning with A (i.e., 105.1A).
.02edit type0;2SET OF CODES1:SYSTEM EDIT
2:ON-LINE EDIT
3:BATCH EDIT
4:OTHER EDIT
5:SYSTEM/ON-LINE EDIT
6:DELETED
7:SYSTEM/BATCH EDIT
8:ON-LINE/BATCH EDIT
The type of the edit. SYSTEM edits are edits performed by the system (i.e., date format edits are SYSTEM edits because FileMan will not allow incorrect dates). ON-LINE edits are performed as the user enters input (i.e., some SOURCE OF ADMISSION edits). BATCH edits are executed at the end of a process (i.e., the Data Validity Checks at the end of the Disposition Enter/Modify option). OTHER edits may include data elements that are values calculated everytime the edit is executed and that, by definition of the calculation, will always be correct (i.e., AGE AT ADMISSION must be coded 00D-30D, 01M-11M, or 01Y-99Y).
