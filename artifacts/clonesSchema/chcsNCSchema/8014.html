RCMAS-OSE SIDR HOLDING

8014

description
This file contains an entry for each SIDR holding file created for the Retrospective Case Mix Analysis System - Open System Environment. Each entry documents one occurance of data having been captured into a VMS flat file in preparation for Kermit transfer to the RCMAS-OSE environment.
Fields
#NameLocationTypeDetailsIndexDescription
.01date(+)0;1FREE TEXTThe DATE field is date of SIDR data. For Monthly, the DATE field is the month and year for which the monthly SIDR tape was generated (e.g., May 1993). For Interim, the DATE field is a range of user-specified dates for which the Interim RCMAS option was run (e.g., May 5, 1993-May 12,1993). The DATE field is stored in FILEMAN format. In the examples above, the Monthly DATE field would be 2930500 and the Interim DATE field would be 2930505-2930512.
.02sidr file type(+)0;2SET OF CODESM:MONTHLY SIDR
I:INTERIM SIDR
The SIDR FILE TYPE field is a set of codes representing the SIDR file type (M=Monthly, I=Interim). The Monthly SIDR file is created when the user runs the monthly SIDR tape option to produce the monthly SIDR tape. The Interim SIDR file is created when the user selects to build an Interim SIDR file on-demand.
.03file status(+)0;3SET OF CODES0:FILE NOT READY
1:FILE READY
The FILE STATUS field indicates whether there is data ready for transmission (1=ready to transmit) or that data is being written to the file to be transmitted (0=not ready to transmit).
.04number of sidr records0;4NUMERICThe NUMBER OF SIDR RECORDS field is the number of logical records in the SIDR file. Each patient register number counts as one record.
.05total number of sidr records0;5NUMERICThe TOTAL NUMBER OF SIDR RECORDS field is the number of physical records in the SIDR file. There are six segment records for each patient register number, so the TOTAL NUMBER OF SIDR RECORDS would be the number of patient register numbers x 6.
.06date/time last transmitted0;6DATE-TIMEThe DATE/TIME LAST TRANSMITTED field is the date/time that the SIDR file was last transmitted to RCMAS-OSE. The SIDR file is transmitted to RCMAS-OSE on demand via the RCMAS option.
.07user who built file0;7POINTER3The USER WHO BUILT FILE field is the internal entry number (DUZ) of the user who initiated the creation of the file. For Monthly SIDR files, the USER WHO BUILT FILE is the user who generates the monthly SIDR tape. For Interim SIDR files, the USER WHO BUILT FILE is the user who requests to build the Interim files via the RCMAS option.
.08date/time file built0;8DATE-TIMEThe DATE/TIME FILE BUILT field is the system date/time that the SIDR file is created.
.09file delete flag0;9SET OF CODES1:DELETED
The FILE DELETE FLAG is a set of codes indicating whether the file containing the actual SIDR data has been deleted from the system (1=DELETED).
.1division(+)0;10POINTER40.8The DIVISION field is the division that the SIDR file was created for (pointer to file #40.8). The DIVISION field will be automatically set to the user's division when the SIDR file is created.
1directory path/file name(+)1;1FREE TEXTThe DIRECTORY PATH/FILE NAME field is the name of the directory path and file name of the file that contains the actual SIDR data.
