description
This file contains the Site definable Clinical Interface Parameters. There can only be one entry per site, and the parameters can be edited from the Clinical Interface Management Menu. In addition there are options to monitor Desktop useage, and to purge certain Desktop logs. This file serves to limit the Desktop functions, but is not required for Desktop functionality.

#NameLocationTypeDetailsIndexDescription
.001number11Enter a number between "1" and "1".
.01site(+)0;1POINTER4BThis field is the name of the Site, that the parameters affect.
1.01term lock-hours1;1NUMERICThis field allows the site to set the maximum number of hours that a user can lock a terminal from the Desktop or Menu. If it is not set or set to 0, it will disable all terminal locking.
1.02term lock-users1;2NUMERICThis field allows the site to set the maximum number of terminals that can be locked from the Desktop or Menu. If it is not set or set to 0, then there will be no limit. If the site needs to disable terminal locking, then set the field 'TERM LOCK-HOURS' to 0.
1.03term lock-each user1;3NUMERICThis field allows the site to set the maximum number of terminals that a given user can lock from the Desktop or Menu. If it is not set or set to 0, then there will be no limit.
1.05view files-per user1;5NUMERICThis field allows the site to set the maximum number of View Text entries that each user can create using the CHDEVICE device. If set, then any Desktop View Text marked as 'SYSTEM' will be counted towards this maximum. If it is not set or set to 0, then there will be no limit.
1.1vms editor-access1;10SET OF CODES-1:DISABLED FOR ALL
0:ENABLED FOR ALL
1:USERS WITH 'CHVMSED' KEY ONLY
The Clinical Interface Parameters file 8800, VMS EDITOR-ACCESS field 1.1, allows the site manager to restrict access to the VMS EDITOR (TPU) via the Desktop Actions that interface to it (PTW & VTW). If this field is set to -1, then all users will be prevented from from accessing TPU via the Desktop. If it is set to 1, then only those users who hold the 'CHVMSED' security key will be allowed to access TPU. If it is not set or set to 0 (zero), then all users will have to access to the VMS Editor. Remember, this field only restricts access to TPU through Desktop.
2.01log actions2;1BOOLEAN0:NO
1:YES
This field allows the site to log all users' actions from the Desktop. This log can be reviewed, and purged from the Clinical Interface Management Menu (CH-SITE-MGR option).
