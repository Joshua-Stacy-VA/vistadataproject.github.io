.001number11This is the unique number which is assigned to a message at the time it is entered into the system. Messages are numbered sequentially, and numbers are never re-used. Numbers appear in the "Query" listing for the message, as @, where is the message number for this message, and is the name of the domain, as defined in the kernel site parameter file. This combination of number and domain name uniquely identifies ALL messages in the mailman network, and editing control is such that no two copies of the message at different domains are different; the message is either identical or missing.

1.6smtp rcpt0;6SET OF CODES0:local recipients only
1:network and/or local recipients
2:relay message
For Network MailMan, this field is a set of codes indicating what type of users are recipients on the mail message. The following codes can be selected for the field: 0 = local recipients only 1 = network recipients(s) and/or mailgroup(s) 2 = this is a Relay message (we are simply a relay point); there are no recipients stored in the MESSAGE File If the selected code is 1, then local users may or may not be recipients. Network Mail Groups are 'expanded' when the message is received at the remote site. If the selected code is 2, then the message is marked for deletion once the message is sent to the next relay point (which may or may not be the final destination). The RCPT TO: (recipient) information is stored in the SMTP Transmission Queue.

7.01smtp from7;1FREE TEXTIf the message is of TYPE=1 (network), then this field indicates the network sender of the message stored as free text. 9incoming message id5;1FREE TEXTIThis is an index to incoming messages from remote nodes on the network. The Message ID of the remote message, as contained in the message header is stored here. This index is used to retrieve remote messages which have been transmitted from other domains. For example, if message 123 from loma-linda has been sent to another site, that other site could inquire against message ID "123@loma-linda", to find out if the message had been received yet. This index is also used to prevent messages from looping around the network. The MailMan receiver will not allow the same message to be received twice at the same site.
