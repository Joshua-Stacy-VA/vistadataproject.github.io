description
The drug clinical screening definitions file (8254.02) consist of all of the information from FDB to perform drug clinical screening checks and reporting. There is no entry in this file it the corresponding entry in the drug file has not been assigned a Primary NDC Number. If a Primary NDC Number exist, information about its Therapeutic Class, route, interactions, allergy classes and monographs, and ingredients are kept in this file. PS DRUG CS DEFINITIONS ien is Dinum to the Drug file (50).

1.05skey rt1;5POINTER8251.3This field contains the smart key Route Code. The smart key Route Code is a two-digit number (with leading zeros removed) representing the route of administration. An SKEY_RT code value is interpreted the same for all SKEY subfields. The SKEY_RT is compatible with the NDDF* one-character (GCRT) and two-character route (GCRT2), forming a one-to-one relationship with each. The compatibility provides both downward and upward compatibility with existing NDDF* data. The SKEY_RT allows the selection of route specific products with or without other SKEY subfield definitions, providing easy searches and easy formulary definitions. If an SKEY_RT for an NDC is assigned zero, CHCS will assign it a numeric value by concatenating each ASCII character of the GCRT field as its numeric equivalent $A(). All NDCs using the same two-character route description will point to the same value. As First DataBank assigns an SKEY_RT to that route description, CHCS will change the SKEY_RT to reflect the assignment. Route of administration displays as part of the identifier of an NDC. The 10- character Route Description (RT) found in the ROUTETBL is the fifth space- delimited parameter to display. The NDC ID is formatted as NNNNN-NNNN-NN GNN (BN) STR RT GCDF. The SKEY_RT on CHCS is a pointer to 8251.3 (ROUTETBL). *NDDF is a Trade Mark of the Hearst Corporation.
