description
This file tracks each task that Orders create. One Order may produce one or many tasks (eg. a medication QID produces four tasks per day). One may think of the posting of a task as similar to an Order Requisition in manual Order Entry. For each task, the two cross references are produced which act as the requisition. Each of the Ancillaries is looking for new requisitions on a regular basis. As the requisition is processed, the cross reference is killed. Finally, the status of the task is updated as appropriate by the ancillary.
