4TPOCS ACTIVECOMPUTEDIn ADM 2.3 the determination of whether or not to produce a TPOCS extract file was based on field 1, now called OBSOLETE - TPOCS ACTIVE. In ADM 3.0 the determination is now based on the Outpatient Itemized Billing field in the Medical Center Division file. The OIB field in file 40.8 is used strictly to determine whether or not to produce TPOCS extract files. The OIB field can only be set for the parent division. Therefore TPOCS will be active or inactive for all divisions that share the same Group DMIS ID. This computed field returns Yes or No. It checks the OIB field in the parent division.
5TPOCS FILENAMETPOCS;4FREE TEXTFilename of the most recent ADM TPOCS extract file produced. This is used for determining the sequence number portion of the next file to be created. TPOCS data for all divisions with the same Group DMIS will be written to the same file.
10CCE RECS SENT TO CCE BY CAPERCCE;1NUMERICRecords sent to CCE with CCE status of "Pending". Total is set for each division at the end of each nightly CAPER process.
11CCE RECS RECEIVED SINCE CAPERCCE;2FREE TEXTRecords received from CCE that pass initial validation checking. After validation of incoming data, each record will have a CCE Coding Status of Received, Released, or Uncertified and this counter will be incremented. This field will be updated anytime CCE is sending valid data and it will be set to zero by the CAPER task. This field is free text because it tracks separate counters for CCE Complete (COMPL), CCE Released (RELEASED), and CCE Uncertified (UNCERT). "COMPL" is incremented anytime the CCE Coding Status is set to Received regardless of whether the incoming data is different from the data that was already in the ADM record.
12CCE RECS RECEIVED BEFORE CAPERCCE;3FREE TEXTThe value in CCE RECS RECEIVED SINCE CAPER is moved to this field each night when the CAPER task runs before the CAPER sets the field back to zero.
13CCE RECS THAT PASSED CAPER CHKCCE;4NUMERICThe records returned to ADM by CCE that went from Received to Complete Status in the nightly CAPER processing.
