.01QUERY MESSAGE WAIT TIME(+)0;1NUMERICBDefines the length of time which will be allowed for the interface to access a lock in an attempt to process an inbound Patient Query message. When the attempt to access a lock exceed this wait time, an error will be logged.
.02ERROR LOG PURGE DAYS0;2NUMERICThe number of days that an error may remain in the DEERS Patient Inquiry Error Log file (8524). After this time, the error will be purged. The default is 4 weeks.
.03COLLECTION FILE PURGE DAYS0;3NUMERICThe number of days that message information may remain in the DEERS Patient Inquiry Collection file (8522). After this time, the message information will be purged. The default is 4 weeks.
.04LOCK NUMBER OF TRIES0;4NUMERICNumber of times that the interface will attempt to access a lock. After the specified number of attempts, an error will be logged.
