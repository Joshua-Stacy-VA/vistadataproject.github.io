.07LOOKUP PARAMETER(+)0;7SET OF CODESF:FORCED LAYGO
L:LAYGO ALLOWED
N:NO LAYGO
O:LOOKUP ONLY
P:PARSE ONLY
This value is used for incoming data to determine which entry in the local database is to be updated. It affects how a lookup is done as follows: FORCED LAYGO = a new entry in the root file will always be created. LAYGO ALLOWED = an attempt will be made to match the incoming data to that already in the local database (using the fields identified as being used for lookup). If the attempt fails, then a new entry will be created. NO LAYGO = an attempt will be made to match the incoming data to that already in the local database as in the LAYGO ALLOWED case, but if the match fails, an error will be generated and no new entry will be made. LOOKUP ONLY = perform the lookup but no storing operations for the root file.
