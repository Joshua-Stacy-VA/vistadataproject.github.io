description
This file stores the data for compliance with the BuMed DQ standard for End-of-Day compliance, which specifies that clinics are to close out all appointments the day of the appointment (24/7 clinics have until 0600 the following day to close out appointments). EOD compliance is tracked for location types 'C' and 'S' (clinics and APUs) for clinic visits (excludes telephone consults and IBWA locations). The data in this file is populated automatically by the GSEODTSK routine, as the GS POPULATE EOD COMPLIANCE option, tasked to run daily at 0002; it should not be modified by other routines or by users. To minimize the disk storage requirements, there is no 'B' cross reference associated with this file; reports referencing EOD compliance should jump to this file from the PATIENT APPOINTMENT file (#44.2). The daily process that populates this global, evaluates EOD compliance for appointments from the previous calendar day. Appointments that do not have a GS EOD COMPLIANCE record were entered into CHCS and backdated using EOD processing.
