# DENTV DRM NO PCE DELETE

This parameter controls whether or not PCE will be automatically updated when the user deletes transactions in DRM Plus.  If set to yes, then the system will attempt to delete the transactions (CPT codes, ICD-9 codes and Providers).  If set to no, then the user will be prompted to manually &#x27;fix&#x27; the PCE data.

--- | ---
ID | 488
Summary | Don&#x27;t update PCE with deletions
Value Type | yes
