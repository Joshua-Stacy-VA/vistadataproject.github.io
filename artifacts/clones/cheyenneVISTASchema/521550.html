This is used to identify the nodes that are on a system. This allows various utilities to avoid vendor specific code.
Fields
#NameLocationTypeDetailsIndexDescription
.01node(+)0;1FREE TEXTB This is a node on the site's computer system. The name must be three uppercase characters.
.02working uci(+)0;2FREE TEXT This is the UCI within the system node that will contain the routine(s) that actually perform the needed tasks. The UCI name must be three uppercase characters.
.03time differential0;3NUMERIC This is used by the module that coordinates the time on all nodes of the site's system. This allows the site to stagger the times of the nodes, for example, allowing the print server to run two minutes behind the compute server. This is the number of SECONDS that will added (or subtracted if negative) from the time of the node on which the time update is run.
.04master node/uci0;4BOOLEAN1:YES
0:NO
 If a master node/UCI is defined, tasks that update all nodes and are queued will be queued to initially start on the master node/UCI. Only one node/UCI can be designated as the master node/UCI. You must have a WORKING UCI defined for a node to be able to designate the node as the master node/UCI. If TaskMan is running only on one node, that node should probably be designated as the master node/UCI.
.05unrestricted time change0;5BOOLEAN1:YES
0:NO
 This is used to allow the site to notify the time change routines that the date/time on this node can be set back at any time of the day. Unless this is set to 'YES', the date/time on this node can only be set back between 2am and 3am.
1description1;1FREE TEXT This is a brief description of the system node, such as "PRINT SERVER".
