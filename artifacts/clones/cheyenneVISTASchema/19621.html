DSIV VISTA IMAGING QUEUE TRACKING
number
19621
location

description
This file contains the transaction IDs sent to VistA Imaging via the VistA Imaging Import API. Once the Vista Imaging Background Processor processes the queued requests, it will invoke the call back routine to update this file with the status of the import request.

.01name(+)0;1FREE TEXTBThis is a unique identifier used by VistA Imaging. For DocManager, it is of the form DSS; or DSS;<-> For all other DSS applications, it will be in the form DSS;
.02queue0;2NUMERICThis is the queue number passed back to the calling application after invoking the MAG4 REMOTE IMPORT rpc which imports an image into VistA Imaging.
.03status0;3SET OF CODES0:Error
1:Successful
p:Pending
This is the current status of this import request to VistA Imaging. p - Pending - this means the request has been successfully placed in the VistA Imaging queue, but the background processor has not processed the request and has not called the callback status routine 1 - Successful - request was successfully processed by the VistA Imaging background processor. There may or may not be associated warning text. 0 - Error - request was processed by the VistA Imaging background processor but was not processed due to errors

There may be a need to keep a history of import requests to VistA Imaging by DSS applications (or one of it partners) which do not have their own external database to track this information. The DSS application DocManager does not have need of this as it has it own SQL database where it keeps track of VI import requests. This field will be updated by any non-DocManager GUI application which invokes the RPC: DSIV DM GET STATUS.
