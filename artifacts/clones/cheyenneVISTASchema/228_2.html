description
This file contains all the data needed for the Discus Dental Treatment Planning (TP) COTS product and Dental. There are four data types in this file. The Discus application usually stores these data types in separate tables. The type of record is stored in the .29 field. Description of field data types ------------------------------- No single transaction will have data in all fields. Which fields get populated depends upon the record type. Some of the fields in the .02 - .29 range may be populated in the different data type records. In general (although not exclusively): TP (or DRM) transaction data are in fields .02 - .29, 1.01 - 1.999 Periodontal exam data are in fields 2 - 2.999 Peridontal screening and recording data are in fields 3 - 3.999 Head and Neck (or soft tissue) data are in fields 4 - 4.999 Regarding output transforms on Boolean fields --------------------------------------------- If $G(DENTFMTP) then the call is coming from a M routine for the purposes of treatment planning software. That software uses -1 for true or yes and 0 for false or no.

Once a transaction type record has a status of completed, then a new style cross reference will stuff a value of P in this field and set up the AXMIT cross reference. When the batch HL7 processor runs, if it successfully transmits this record, then that batch process will change this flag to transmitted. If the HL7 message could not be generated because of a data error, then the field will be set to E for error. If a record is deleted (i.e., Date Deleted field is entered), then this status will be changed to cancelled if the current status is pending or null. If the status is transmitted, then this field will be changed to pending so that the batch processor can transmit a delete HL7 message.
