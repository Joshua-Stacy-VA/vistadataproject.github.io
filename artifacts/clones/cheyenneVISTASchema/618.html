This file contains CURRENT data used by the In-Patient Features Option of the Mental Health System.
Fields
#NameLocationTypeDetailsIndexDescription
.01ward(+)0;1POINTER42BThis field is a pointer to the Hopsital location file. It contains the ward name as found on the G&L.
1synonym(+)0;2FREE TEXTSYNThis is a free text field used to print abrreviated names of wards for various reports. It is suggested that this be less than 15 characters and clearly distinguish wards.
2face sheet order(+)0;3NUMERICAFSThis field specifies the order of the wards on various reports, for example on the Bed Census report. Face sheet order must be present and each ward must have a unique number. It is suggested that these numbers not be sequential but separated by at least 5 so later additions or movements can be easily accomplished.
2.2list subtotal(+)0;4BOOLEAN1:YES
0:NO
This is a set field that casues a census sub-total to be displayed on the Bed Census "face sheet". This is useful to indicate census over several wards ie 3 "acute" units.
2.3zero subtotal0;5BOOLEAN1:YES
0:NO
This set field specifies whether the sub-total count should be reset to 0. This is used to either show serveral distinct sub-totals or to keep a running sub-total.
2.4subtotal title0;6FREE TEXTThis free text field is displayed as the title of any sub-total displayed on the Bed Census "face sheet". If no sub-total is displayed this field is ignored. If a sub-total is requested and this field is empty, the default "sub-total" will be displayed. It is suggested that all subtotal titles contain the phrase "sub-total" eg Acute sub-total.
2.5line feed0;7BOOLEAN1:YES
0:NO
This field if set will add a blank line following a ward on the Bed Census "face sheet". This is for ease of reading only.
