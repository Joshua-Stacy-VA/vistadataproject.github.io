description
The "Bingo Board" application is designed to list orders from inside and outside CPRS. It is currently being expanded to also allow applications like a Pharmacy BB display, and for remote display where a medical center provides night/off hours service to other MCs. It is expected that it can be expanded and/or utilized in its original form for any application locally decided. The original was designed essentially to display non-verified nursing orders and is a hard addressed printer-assignment throught the M -> VMS device system. It now also contains a broker-based application (from Portland) and a web-based application (from Murfreesboro). We are expecting to pack all these into one package for interested sites.

10.4use pharm wards file?PS;4BOOLEAN0:NO

Originl Implementation: The Inpatient Pharm BB originates from Portland VAMC. It was set up to assist pharmacy with the gradual implementation of BCMA. Only wards that were starting the BCMA process were included. However, with the interest by other sites in the country and not necessarily wanting the ward listing restricted, this field has been included into Ann Chu's BB parameter system. So, Use Pharm Wards File? 0 means that ARHCOPB (and BC) will list any ward. 1 means that ARHCOPB (and BC) will use only wards listed in 648041.
10.5scroll pharm bb?PS;5BOOLEAN0:NO
1:YES
This pertains strictly for the inpatient (and outpatient) pharmacy BB system. The intent of the board is to display non-verified orders (for inpatients from 53.1, and for outpatients from its OPT shadow), the oldest urgency on top, and then after that, the oldest non-urgent order on top. Even thought the display can be compressed, it can be larger than the displayable screen. The Pharm BB system is set to a 30 second refresh. So, SCROLL PHARM BB: 0 (NO) means that the display will always refresh from the oldest to the newest orders in 53.1, and will not scroll up beyond the oldest orders. 1 (YES) means that a refresh will scroll to the next screen, if there is the list of orders is longer than the current display screen.

10.7verifications screenVERPROC;1SET OF CODES0:NOT APPLICABLE
1:NURSE VERIFICATION ONLY
2:CLERK VERIFICATION ONLY

Portland VAMC has wards that have mixed responsibilities for verification of orders. Ward secretaries are responsible for certain labs on treatment. However, under the original version of the Indianapolis enhancement, an order verified by a clerk or nurse will be removed from the board. This parameter is meant to assist sites (like Portland) where clerks may act on orders, there behavior on verification will be maintained, and nurses, additionally will view those orders. An order at Portland will only be cleared when the clerk AND the nurse have verified. Hence, the set of codes in the parameter file. jim bell 4/5/01
