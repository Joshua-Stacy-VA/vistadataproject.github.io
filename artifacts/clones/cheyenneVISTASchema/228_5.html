description
This file records data that is pertinent to the Dental Central Office for managing fee basis dental treatment. This file will eventually be replaced by the VA's redesign of the fee basis package. Dr. Zeller, Director for Clinical Informatics, Dental VAHQ, considers this file part of the clinical medical record. As such, no entries may be deleted. They will be marked as deleted and will be screened from being viewed via DRM. IA for pointer to the PATIENT (#2) file for field .04 is 10035. IA for pointer to the NEW PERSON (#200) file for fields .03 and .1 is 10060.

This field contains the status of the Dental Fee Basis HL7 transmission. "P" means the transaction is pending submission to AAC. "T" means the transaction has been transmitted. "C" means the transaction was marked as deleted before it was transmitted. If a record is deleted AFTER being transmitted, then this field will be changed back to "P" to be resubmitted. If the HL7 message could not be generated because of data errors, then this field will be set to "E" for error.
