********************************************************************* Generic Progress Note Title File. Titles, or subtypes, under which a progress note may be filed. Every title must be associated with a Generic Progress Note Type. *********************************************************************
Fields
#NameLocationTypeDetailsIndexDescription
.01name(+)0;1FREE TEXTBThe title given to a progress note, to clarify its content upon review.
.02type of progress note(+)0;2POINTER121.1CEvery progress note must be associated with a type; titles provide a "subtype" for notes, further clarifying their content.
.03uppercase title0;3FREE TEXTD
.04inactive0;4SET OF CODES1:INACTIVE
 This field is used to 'inactivate' a Progress Notes title. The user will not be permitted to add notes with an inactive title but the notes with an inactive title will print as before.
.05security key0;5POINTER19.1This field contains a security key associated with this TITLE. It is used to prevent users from authoring new Progress Notes of this TITLE if they do not hold this security key. This field is editable and has no impact on Progress Notes data.
.06no cosignature required0;6SET OF CODES1:NO COSIGNATURE REQUIRED
If this field contains a '1' or 'NO COSIGNATURE REQUIRED' than all new notes created under this TITLE will not require cosignature, even if the user holds the 'GMRPCS' key. This is helpful if the site has specific TITLES that never require signature and the user enters some notes that do
1note counterCTR;1NUMERICThis field is used to keep a count of the number of notes associated with this TITLE. It will be used to aid the sites in structuring the Document Definition file (#8925.1) in TIU. routine (^GMRPCTR) will loop thru and increment this field for each SIGNED note in Generic Progress Notes file #121.
1.01tiu doc class1;1POINTER121.3This field is required in order to set this TITLE up in the DOCUMENT DEFINITION file (#8925.1) in TIU. The Document Definition Heirarchy for progress notes needs to be defined in the PNs TITLE file #121.2 prior to running the conversion.
1.02no edit class1;2SET OF CODES1:NO EDIT
This field is used to designate certain DOCUMENT CLASSES that must be in place in TIU. Primarily these are the TITLES associated with the CWAD warning system. These doc classes are exported by GMRP*2.5*44 and cannot be changed.
1.03ien in tiu1;3NUMERICThis is the IEN of the entry corresponding to this TITLE in TIU 8925.1. It is not a pointer because patch GMRP*2.5*44 does not require TIU be installed.
