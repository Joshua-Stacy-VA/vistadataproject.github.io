description
The Quality Improvement Checklist Question file stores the pertinent data about each question which is answered on the QUIC semi-annual survey. Fields include the question text, type of response expected, parameters of the expected answer, and the surveys in which the question is asked. This file arrives at the VAMC with data, and SHOULD NOT be altered locally.

22.1upper limit12;2NUMERICQUIC questions which have the OUT-OF-RANGE FLAG set to "1" may have the UPPER LIMIT field set. If the UPPER LIMIT field is set, it will contain the highest expected answer. At transmission time, questions will be reviewed to determine if the answer is beyond the expected limit. The individual transmitting the data will be warned if answers exceed the UPPER LIMIT (this is usually 100%).
22.2zero flag12;3BOOLEAN1:YES
0:NO
QUIC questions which have the OUT-OF-RANGE FLAG set to "1" may have the ZERO FLAG field set. If the ZERO FLAG field is set, it indicates that zero (0) is an unacceptable answer for this question. At transmission time, questions will be reviewed to determine if the answer equals zero (0). The individual transmitting the data will be warned if this occurs and will be given an opportunity to edit the answer. If the individual transmitting the data does not heed the warning and chooses to send the erroneous data, the software makes a conversion. If the answer to this question equals zero (0), the software converts the answer to "ERR" prior to transmission. This prevents the "0" answer from skewing the national and RAM group averages.
23non-applicable flag13;1BOOLEAN1:YES
0:NO
This flag will be set to "1" if the answer to the QUIC question is to be converted to "N/A" prior to transmission if the facility answered the question with a "0". This would be the case when a facility performs a specific service/function, however, during the survey period, there was no request for that service/function. This prevents the "0" answer from skewing the national and RAM group averages.
23.1affected entries13;2FREE TEXTThe AFFECTED ENTRIES field is to be used in conjunction with the NON-APPLICABLE FLAG. If the NON-APPLICABLE FLAG is set for a question and if a facility answers zero (0) for that question, prior to transmission that answer is converted to "N/A". However, there may be other parts of the question which should also be marked as non-applicable if the "core" part of the answer was zero. For example, if the question pertains to the "number of procedures" performed, and a facility answers zero, then logically, there could have been no deaths or a mortality rate associated with "zero" procedures. The AFFECTED ENTRIES field contains the internal entry number(s) of the question(s) which will be converted.
