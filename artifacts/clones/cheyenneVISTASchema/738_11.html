Will the answer to this variable be "user supplied"? The answers to some variables are stored in the file when the software is installed (i.e., beginning and ending dates for the "current" survey). The answers to other variables must be supplied by the user before an option can be "automatically" queued to print. If this variable is answered by the user, enter "1". If the variable does not have to be answered by the user enter "0".
2display order0;3NUMERICADSPThe display order field controls the order in which the variables will be presented to the user for answers.
3category(+)0;4SET OF CODESS:SINGLE ENTITY
M:MULTIPLE (ARRAY)
In order to "pre-answer" each required variable before queueing the report, the software must know if the answer to a variable is a single entity (i.e., report beginning date) or an array (i.e., a number of laboratory tests). If the variable is to be answered with a single entity, the category field will be set to "S". If the variable is to be answered with multiple entities, the category field will be set to "M".
4type(+)0;5SET OF CODESP:POINTER

This field identifies the type of answer which will be accepted for each variable. If the variable will store a pointer number, enter "P". Enter "N" if the variable will contain numeric data. A "D" refers to a date in FileMan format, and an "S" refers to a set of codes.
