description
The Remote Order/Entry Parameter File contains a number of fields that give the ADPAC's at each local facility the ability to customize portions of the system for use at their stations. Definitions of each parameter and recommendations for the settings to be used are included in the Parameters section of the ROES V2.0 Technical Manual. Data definitions are distributed with the ROES. No data is distributed.

#NameLocationTypeDetailsIndexDescription
.01station(+)0;1POINTER4BThis field is a pointer to the Institution File for each station that sets up parameters for the Remote Order/Entry System.
.02allow free text address0;2SET OF CODES1:ALLOW FREE TEXT ADDRESS
0:DON'T ALLOW FREE TEXT ADDRESS
If this field is answered 'YES', in the Enter/Edit Patient Orders option the user will be asked if a new address should be entered for a patient. If a new address is entered, the new address will be transmitted to the DDC for update of the DDC database. The DHCP patient file at the station will not be changed. The patient should be sent to MAS to update the station database. If 'NO' is entered, the address in the DHCP patient file will be used for the order.
.03transmission queue options0;3SET OF CODESI:immediate queue of batch
S:standard queue time for batch
C:choose queue time for batch
A:auto-queue batch
This parameter controls the queue time for transmission batches. If an 'I' is entered, batches will be queued for transmission immediately upon closing. If an 'S' is chosen, the user will be asked to set a parameter defining the time of day for queueing transmission batches. If a 'C' is entered, the user closing a batch will be asked for a date and time to queue the batch. If an 'A' is entered, the batch will be queued automatically every day at the time set in the parameter 'Standard Queue Time for Batch'. If no entry is made in this field, immediate queueing will be assumed.
.04standard queue time for batch0;4DATE-TIMEIf the System Manager has chosen to set a standard time for queueing of transmission batches to the DDC, this field should hold the time of day for that transmission. Any batches that have been closed will automatically be queued for transmission every day at this time.
.05multiple station orders0;5SET OF CODES1:MULTIPLE STATIONS
0:ONE STATION
This field indicates whether or not a field facility wishes to allow ROES orders to be created for multiple station numbers.
.06allow multiple approval0;6SET OF CODES1:ALLOW MULTIPLE APPROVAL
0:DO NOT ALLOW MULTIPLE APPROVAL
If the user chooses to allow multiple approval of orders, the Approving Official will be able to select a group of orders to approve without reviewing the display screen for each order.
.07approval process0;7SET OF CODESS:supervisors only
A:audiologists and supervisors
This field indicates whether a station wishes to allow both supervisors and Audiologists to be able to approve orders or if they only wish to allow supervisors to have approval privileges. If no entry is made in this parameter, only supervisors will be able to approve orders.
.08eligibility edit capability0;8SET OF CODES0:NO EDIT OF DHCP ELIGIBILITY
1:EDIT OF DHCP ELIGIBILITY BY SUPERVISOR
2:EDIT OF DHCP ELIGIBILITY BY ANYONE
This parameter controls users abilities to edit the eligibility for a patient. If set to '0', the DHCP eligibility determined from the database cannot be edited. If set to '1', the DHCP eligibility can only be edited by a supervisor. If set to '2', the DHCP eligibility can be edited by any user.
.09display errors on menu entry0;9SET OF CODES0:DO NOT DISPLAY ERRORS
1:DISPLAY ERRORS
If this parameter is set to 'DISPLAY ERRORS', all users signing on to ROES will see a list of orders in the error status.
.1order delivery routing0;10FREE TEXTThis field contains the free text routing symbol of the service requesting a regular mail response to correspondence or product delivery from the DDC.

.12issue approval0;12SET OF CODES0:ANY USER CAN ISSUE
1:AUDIOLOGISTS & SUPERVISORS

.13psas eligibility determination0;13SET OF CODES0:ALLOW REQUESTS TO PSAS

This parameter determines whether electronic requests may be sent to PSAS for eligibility determinations for patients whose eligibility cannot be determined from the DHCP patient database.
