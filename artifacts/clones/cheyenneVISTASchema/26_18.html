description
This file contains the configuration parameters for the Patient Record Flag module of the Registration package. The file should contain only one record numbered "1". Modifications to the file should only be done through the Patient Record Flag Configuration [DGPF RECORD FLAG CONFIGURATION] option.

This field is used to enable or disable the Unsolicited Observation Update (ORU~R01) HL7 interface for the Patient Record Flag module.
3prf hl7 query status(+)0;4SET OF CODES0:INACTIVE
1:DIRECT
2:DEFERRED
This field controls the PRF HL7 query (QRY~R02) interface. Setting the value to '0' disables the interface. Setting the value to '1' enables the interface using a VistA HL7 'direct' connection. Setting the value to '2' enables the interface using a VistA HL7 'deferred' connection.

5prf hl7 auto retransmit period0;6NUMERICThis field represents the number of days to wait before an auto retransmission of Rejected HL7 messages should occur.
6prf phase 2 activation0;7DATE-TIMEThis field contains the date that is used to restrict access to specific Patient Record Flags functionality in production environments until all sites have installed the PRF Phase II software.
