.02status0;2SET OF CODESC:COMPLETE

This field is used to indicate the status of the message. A message that has not been sent will have a status of pending. A message that is currently being sent will have a status of incomplete. If an error occurs while sending the message the status will be set to error. A Complete status indicates that the message was successfully sent to the recieving system.

.06bypass obs0;6SET OF CODES1:BYPASS OBS
This field when set indicates that when sending the message the OBS should be bypassed and send to the IP address of the destination.
.07patient0;7POINTER2This field tracks the patient for OREQ message.
.08number of retries0;8NUMERICThis field is used to determine how many times the system has tried to send the message to the receiving site.
.09message type0;9SET OF CODES0:OREQ
1:NOT OREQ
2:MUPD
This field indicates whether the message is an OREQ type message or not. The field is set to 0 for OREQ messages, set to 2 for MUPD messages and set to 1 for non-OREQ messages. When an OREQ message reaches completion, i.e. a corresponding FACK message is received, this field will be set to 1.
