name
LPO PHARMEX DEVICE
number
8221.1
location
^PSI(8221.1,
description
By default it is assumed that all Lexmark printers include the PharMex SIMM chip and no other printer brands include the PharMex symbol set. This file was created to allow sites to override these assumptions.
Fields
#NameLocationTypeDetailsIndexDescription
.01device(+)0;1POINTER3.5BBy default it is assumed that all Lexmark printers are Pharmex enabled and all other devices are not. This file allows you to override this assumption.
.02pharmex capable0;212Indicates if the printer supports the PharMex symbol set.
1pem0;3FREE TEXTOnly devices with a sybtype that starts with P-LEXMARK are allowed to print a PEM (patient education monograph) with each label it is assumed that all Lexmark printers will print the PEM. This field provides that ability to override that assumption. The most common scenario for setting the PEM field to NO is for refill pharmacies that use Lexmark printers. It is possible to have a Lexmark printer with an 8 1/2 X 11 tray and a half-sheet tray. Just set up two different entries in the Device file to reference the same physical device with two different subtypes.
2flush each script0;412To reduce paper jams in Lexmark printers, the paper path is cleared before printing the next label. The down side is that it is much slower.
3reverse orientation0;5SET OF CODESO:NO
1:YES
Send the escape sequence to a Lexmark printer for Reverse Portrait. This allows the site to put label stock in backwards from the normal feed direction. Some sites have heard this reduces paper jams. The experimentation at TMSSC shows that Reverse Portrait orientation dramatically increases paper jams.
4multiple labels per stock0;6FREE TEXTEnter Yes (or 1) to enable the printing of multiple prescription labels per sheet of label stock. Enter No (or 0) to disable multiple prescription labels per sheet and print only 1 prescription label per sheet. When set to YES, the system will print one patient's prescriptions per sheet of label stock.
