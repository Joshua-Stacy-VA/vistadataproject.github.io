description
The Execute Code file is a collection of MUMPS code used by the software to perform specific functions. None of the supplied entries should be deleted. The file defines specific information such as name, the MUMPS code, interval of purge control, type of code, and subscript name(which should never be altered). ATT: PROGRAMMERS For test results which are system calculated: The Execute Code MUST be the same as the line tag in ^LRUTITR which is called to perform the calculation. (ie. LDLCALC - The code says D LDLCALC^LRUTITR.)

The type of code as categorized by the set of codes. Identity, verfication, accession, loadlist, or edit. DELTA CHECK: The delta checks are used in results entry to check if result has changed from previous certified result over an pre-set limit (DELTA-VALUE) according to certain algorithm. Delta checks are defined in LAB TEST file as a sub-field of SITE/SPECIMEN field. EDIT: This type of execute code is entered in the Edit code field of the LAB TEST file (#60). The microbiology subscripted tests use edit codes to define the fields needing editing for result entry. Parasitology has different resulting needs than Culture & susceptiblity EXECUTE: These are codes that are used in various lab programs. They should not be changed (with the exception of "SYSTEM STATUS" for site whose operating system does not use %SS). TRANSFORM: Input transforms are used to calculate a result (LRRE) according tof some formula from raw data that is asked user or available from some other source. VERIFICATION: Used in the verification code field in the ACCESSION file (#68). Determines the QC method (either Bull's algorithm or Multi-rule.) to be used in that accession area.
