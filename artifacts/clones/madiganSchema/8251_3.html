ROUTETBL - RROUTED1 The Route Code Description Table uses the 1-byte Route of Administration Code to provide a 10-byte route of administration abbreviation, an alternative 2-byte route of administration code and a 40-byte route of administration description. The ROUTETBL is associated with the GCNSEQNOTBL through the GCRT field. The Internal entry number (D0) is the SKEY_RT described below: The smart key Route Code is a two digit number (with leading zeros removed) representing the route of administration. An SKEY_RT code value is interpreted the same for all SKEY subfields. The SKEY_RT is compatible with NDDF* one character (GCRT) and two character route (GCRT2), forming a one to one relationship with each. The compatibility provides both downward and upward compatibility with existing NDDF* data. The SKEY_RT allows the selection of route specific products with or without other SKEY subfield definitions, providing easy searches and easy formulary definitions. If a SKEY_RT for an NDC is assigned zero, CHCS will assign it a numeric value by concatenating each ascii character of the GCRT field as its Numeric equivalent $A(). All NDCs using the same 2 character route description will point to the same value. As First DataBank assigns a SKEY_RT to that route description CHCS will change SKEY_RT to reflect the assignment. *NDDF is a Trade Mark of the Hearst Corporation. Description of the FDB Smart Key. The Smart Key is a 24 numeric digit field assigned to each NDC as follows: Col Col Description MNEMONIC # of digits 1 - 2 Generic Therapeutic Class SKEY_GTC 2 3 - 6 Specific Therapeutic Class SKEY_STC 4 7 - 11 Generic Name SKEY_HICL 5 12 - 15 Drug Strength SKEY_STR 4 16 - 18 Dosage Form SKEY_DOSE 3 19 - 20 *Route of Administration SKEY_RT 2 21 - 23 Package Size SKEY_PS 3 24 - 24 Unit Dose - Unit of Use SKEY_UDUU 1 (0=Neither,1=Unit Dose 2=Unit of Use)

.02RT0;2FREE TEXTVRoute Description (RT) is the actual narrative description of the normal method by which a drug is administered. The current range of descriptors includes oral, topical, injection, etc. This field is a free format field. However, the data has been generated from a table to provide consistent spelling forms. A one byte code (GCRT) is available for applications where the description is transparent to the user. There is also a two byte code (GCRT2) which is a meaningful abbreviation.
.03GCRT20;3FREE TEXTCRoute of Administration Abbreviation (GCRT2) is an alternative 2-byte route of administration code. It is a meaningful abbreviation.
.04GCRTDESC0;4FREE TEXTDRoute Code Interpretation (GCRTDESC) is a 40-byte route of administration description. GCRTDESC is the actual narrative description.
.05SYSTEMIC0;5SET OF CODESS:SYSTEMIC
A Systemic Route Indicator (SYSTEMIC) of "S" indicates that the route description is systemic and not route specific. If the systemic routes are transformed, possibly replacing their values with an asterisk, then the algorithm can be simplified to look for equal values. The systemic indicator is also defined as an an 's' value in the route description table. However, the indicator is a First DataBank suggestion. * Systemic routes include any route commonly used to achieve appreciable blood levels.
