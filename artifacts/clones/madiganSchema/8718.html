The Lab Collection List Batch file contains all the data for each collection list batch. The following are non-fileman cross references: Non-FileMan Cross Reference Collection Pool Xref: created at Nightly cleanup and collection list log-in (LGC). Killed off during GCL, UCL, LAG. ^LRO(8718,"APOOL",LRDIV,LRWD,LRRMSEQ,LRBED,LRNAME,LRAA,LRAN)=LRPTN ^LRO(8718,"AOPEN",LRDIV,LRBATCH)=LRODT ^LRO(8718,"ACLS",LRDIV,LRBATCH)=LRODT_U_LRCLDT ^LRO(8718,"ABAT",LRPTN,LRBATCH,LRWD)="" ^LRO(8718,"ABAC",LRPTN,LRACP,LRBATCH)="" LRAA = accession area pointer #68 LRAN = accession number LRBED = patient's bed (sub of room in #42) LRDIV = pointer to division #40.8 LRNAME = patient's name (first 25 characters) LRPTN = pointer to patient #2 LRRMSEQ = room number sequence (sub of #42) LRWD = ward location pointer #42 LRACP = accession pointer #66 LRODT = collection batch generation date LRCLDT = collection batch close date LRBATCH = batch number

#NameLocationTypeDetailsIndexDescription
.01BATCH(+)0;1NUMERICBThis field contains the number which designates each sequential batch generated for a collection list.
.02DIVISION(+)0;2POINTER69.91This field indicates the division in which a collection list batch is generated. For example: Portsmouth, Langley, Eustis.
.03BATCH STATUS(+)0;3SET OF CODESO:OPEN
C:CLOSED
If this field is set to "O", the collection list batch is designated as open. If this field is set to "C", the batch is designated as closed. If open, the batch contains accessions which have not been logged-in, cancelled, or returned to the collection pool and may have new accessions added to it. If closed, the batch contains only accessions which have been logged-in, cancelled, or returned to the collecton pool and may not have new accessions added to it.
.04DATE/TIME BATCH GENERATED(+)0;4DATE-TIMEThis field indicates the date/time a collection list batch is generated.
.05DATE/TIME BATCH CLOSED0;5DATE-TIMEThis field indicates the date/time a collection list batch is closed. A batch is closed via resolution of all accessions on the batch through log-in, cancel, or return to collection pool.
.06OUTSTANDING BATCHES/DIVISION0;612If this field contains a "1", it indicates that there are one or more collection list batches with a status of "open" for the user's division. If this field contains a "0", it indicates there are none.
1WARD1;0MULTIPLE8718.01This is a multiple field which contains attributes for a ward which may have orders for a collection list batch for a division. If the ward does not have any orders for the collection list, the ward number will be listed under a heading of "No orders selected for the following wards".
