description
This file holds uneditable records of all transactions for controlled substances that are being tracked via the Narcotics Inventory System. Each transaction will be assigned a unique sequence number. The control node for this number will be ^PSN(54.4,"*","ID"), a non compatible node with Fileman. The node will store the last number used, and will be incremented, within each day, by one. The format of the sequence number will be a 7 digit Fileman date concatenated with a sequential 5 digit number, starting at 00001 for each day. Examples: 295070100001, 225070100002, 295070100003, 295070200001, etc. The purpose of the sequence number is to identify the exact order in which transactions, in this file, are completed.

11REFERENCE NUMBER(+)0;8FREE TEXTARFNThis field is a variable pointer type field. It is set through hard code to reference entries in other files that have generated a transaction log entry. The first $P("-") reflects the type entry in the other files, eg. RX=Prescription file (52), IS=Issue file (54.3), AD=Adjustment which points back to this file and ST=Supply transaction (54.5). The second $P("-") is the .01 or name field in the other file. The .01 field was used because fileman would not support variable pointers and there was not really a need to have a hard relation between this file and the others as this file is an audit log type file only and is used to record an audit trail of transactions in other files only. The 'CRFN' x-ref will only show up on prescription that have been triggered by the entry of a new, renew, or refill prescription action and will be killed out when the quantity field (DD(54.4,4,0)) is filled in.
