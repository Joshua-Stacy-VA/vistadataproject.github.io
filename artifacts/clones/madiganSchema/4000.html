description
This file stores the transaction types that the GIS will recognize. Transaction types are used to identify a class of transactions. They contain the information necessary to indicate the format of the data and the destination of the message. For outgoing transactions there are two kinds of transaction types: parent and child. A parent contains no routing or format information and is used as the name called from an application program. A child contains full formatting and routing information. The transaction type is also used when processing an incoming message. No parent child relationship exists for incoming transaction types. The destination for an incoming transaction type is a script. This script contains the information necessary to parse the message and store it in the database.

.12ACKNOWLEDGE EXPECTED0;1212This field is used for outgoing transactions to indicate whether or not an acknowledge is to be expected. .13FORMATTER PROCESS TIME0;13FREE TEXTIf this field is non-null, it will indicate a time at which a transaction of this type should be processed. This field will remain as the original text and not be transformed to a date. This value will be used as the implicit value of the 4th parameter of the ^INHF call. Placing a value in the 4th parameter will override this value. Leaving this blank equates to NOW. An entry of STAT will force the transaction to the top of the queue. Examples of valid entry are: T-3@1030, NOW-2H, etc.
.14FORMAT PRIORITY0;14NUMERICThis field is used to prioritize messages as they are processed by the format controller. Valid values are from 0-10 with 0 (default) being the highest and 10 being lowest. If no priorities are assigned to any transactions, or if they all have the same priority, they will be processed in the order received.
.15OUTPUT PROCESS TIME0;15FREE TEXTIf this field is non-null, it will indicate a time at which a transaction of this type should be processed. This field will remain as the original text and not be transformed to a date. It should be a relative value such as T@0500 or T+1@0200 or NOW-2H. A value of STAT will force entries to the top of the queue. Leaving this blank equates to NOW.
.16OUTPUT PRIORITY0;16NUMERICThis field is used to prioritize messages to the output controller. Valid values are from 0-10 with 0 (default) being highest and 10 being lowest. If no priorities are assigned to any transactions, or if all transactions have the same priority, they will all be procssed in the order received.
