description
This file contains the definitions for an action bar. Each element on an action bar has its own entry in this file, as does each bar as well.

3type(+)0;3SET OF CODES1:ACTION

Action Bars are Lists of Actions, any one of which may be chosen, and which when chosen perform a particular action. Button Bars are Lists of Datum, any of which may be chosen, and which answer a question or field implied by the TEXT/PROMPT presented to the end-user. Function Bars are lists of operations and Key Press Sequences which perform that operation. There is no slider on a function Bar, and they define possible key press sequences available to the user. When an entry in this file is KIND=ELEMENT, there are three types of ELEMENT possible: Element on an action Bar -- This entry is intended to be one of the choices available to an end user for some action bar. Element on a button Bar -- This entry is intended to be a datum on a button bar. Element on a function bar - This entry links a keystroke to an operation.

If YES, the slider will appear on the next valid item immediately following the item that was previously the position of the slider. This assumes the previous item continued to be valid. If the previos item no longer is valid, the first valid item in the bar with a DISPLAY ORDER greater than that item's DISPLAY ORDER will be the next position of the slider.
