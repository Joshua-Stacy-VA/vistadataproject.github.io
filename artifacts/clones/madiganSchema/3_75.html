description
This file contains Messages that will be displayed to users as they use the Menu Manager subsystem. The Action for each entry will determine when the message is to be displayed. The Message field contains the actual text that will be displayed.
Fields
#NameLocationTypeDetailsIndexDescription
.01name(+)0;1FREE TEXTBThe name should begin with the registered namespace for each subsystem. For example: XM-NEW MAIL for MailMan.
.02message0;2FREE TEXTThis is the actual Message text that will be displayed to users within MenuMan. It must have at least two consecutive letters in it. This Message should be edited rarely, if at all, to avoid changes in the User interface. The Message will be displayed based on the value of the ACTION field (#.03).
.03action0;3SET OF CODESO:once
L:login
E:either
A:always
This value determines which cross-reference(s) will be set for this Notification Message. This then determines when the messages are displayed within MenuMan.
.04display order0;4NUMERICThis value determines the order in which messages are to be displayed. A Notification Message with a lower Display Order number will be displayed before a message with a higher Display Order. Those messages with no Display Order will be displayed last. Note: The XM-NEW MAIL Display Order should be null.
