This file is used by the LOINC Code Calculation option when deriving LOINC codes for result entry. There are no permanent entries stored in this file.

.01unique id(+)0;1NUMERICBThis field contains a unique identifier associated with the entry in this file. It is temporary and is deleted as soon as the LOINC code calculation process is complete.
.02loinc component(+)0;2POINTER8188.1Select an entry from the LOINC Component/Analyte file. The LOINC Component Name is the name of the component or analyte measured. Examples are "GLUCOSE" for a Chemistry test, "MICROORGANISM IDENTIFIED" for a Microbiology test, or "STUDY REPORT" for an Anatomic Pathology test. A LOINC Component Name must be entered to capture a LOINC Code for a test result.
.03loinc property(+)0;3POINTER8188.2Select an entry from the LOINC Property file. The entry may include the LOINC Property abbreviation and name. The LOINC Property distinguishes between different kinds of quantities relating to the same substance. Examples are MCNC (Mass Concentration), SCNC (Substance Concentration), PRID (Presence or Identity). The LOINC Property must be entered to capture a LOINC Code for a test result.
.04loinc scale(+)0;4POINTER8188.5Select an entry from the LOINC Scale file. The entry may include the LOINC Scale abbreviation, name, and description. The LOINC Scale is the scale of measure for a test result. Examples are QN for Quantitative number results, ORD for Ordinal (1+, 2+, 3+), NOM for Nominal (Yellow, Clear, Bloody). The LOINC Scale must be entered to capture a LOINC Code for a test result.
.05loinc system sample(+)0;5POINTER8188.4Select an entry from the LOINC System/Sample file. The LOINC System/ Sample is the type of specimen on which a lab test is performed. Examples are SER (Serum), SPT (Sputum), BLD (Urine). A LOINC System/ Sample must be entered to capture a LOINC Code for the lab test result.
.06loinc time aspect(+)0;6POINTER8188.3Select an entry from the LOINC Time Aspect file. The entry may include the LOINC Time Aspect abbreviation and name. The LOINC Time Aspect is the length of time for which a property is measured for a test result. It may be a single point in time or a time interval. Examples are PT (Point in Time), 1H (1 Hour), 24H (24 Hours), 2D (2 Days). The LOINC Time Aspect must be entered to capture a LOINC Code for a test result.
.07loinc method0;7POINTER8188.6Select an entry from the LOINC Method file. The LOINC Method specifies the manner or system by which the test is performed. The LOINC Method is entered if it must be specified in order to determine a LOINC Code for the test result.
