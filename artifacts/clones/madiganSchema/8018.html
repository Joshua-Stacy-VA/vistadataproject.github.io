description
This file contains the results of the duplicate patient search. This file is used for statistical reporting.
Fields
#NameLocationTypeDetailsIndexDescription
.01BEGIN RUN TIME(+)0;1DATE-TIMEThe start date/time of the particular duplicate patient search. This value allows the user to specify when to begin a run of a duplicate patient search. When the user accesses the field, the system will suggest a default date/time which has been assigned by the MTF. This default date/time is usually set in the late evening or early morning hours so as to have minimal impact on the operation of the system during business hours. The user can accept the suggested default time or can enter a new date/time to override the suggested date/time default.
.02END RUN TIME0;2DATE-TIMEA user-entered date/time field which allows the user to indicate when the system should stop processing of an uncompleted duplicate patient report run. This field allows the user to stop processing of a duplicate patient report that has not completed by normal business hours in order to minimize the impact this pending report has on the daylight hours operation of the system. If this field is used to actually stop the processing of an uncompleted duplicate patient report run, the processing for the duplicate patient report will resume again at the designated start time previously entered by the user.
.03USER0;3POINTER3The name of the user who is running the duplicate patient report. This user is identified via a pointer to the USER File.
.04TOTAL SEARCHED0;4NUMERICThe total number of records searched using the criteria selected by the user. ALL indicates that all records in the database should be searched and could take a very long time. If using a range, the total number of records searched must fall within the range specified.
.05TOTAL FOUND0;5NUMERICThe total number of potential duplicates found in the search using the criteria specified by the user.
.06TOTAL IDENTIFIED DUPLICATES0;6NUMERICThe number of potential duplicate patients that have been identified as being actual duplicates using the duplicate patient options.
.07TOTAL EXCLUDED0;7NUMERICThe total number of identified potential duplicate patients that have been identified as NOT being duplicates of each other. Based on the search criteria, patients may originally be identified as potential duplicates and later determined by the user to not be duplicates, but in fact patients with similar names and SSNs etc.
.08TOTAL MERGED0;8NUMERICPotential duplicate patients which have been identified as actual duplicates and had their associated data consolidated (merged) into the correct patient record.

.11SEARCH CRITERIA0;11SET OF CODESA:ALL

A set of codes field denoting the search criteria used for duplicate patient searches. The following codes can be selected: A (ALL) Search the entire PATIENT File N (ALPHA NAME) Search the database using the patient's name or a range of names P (REPRINT) Reprint the already completed report R (REGISTRATION DATE) Search the database for potential duplicates of patients that were registered within a range of dates
.12SEARCH BEGIN RANGE0;12FREE TEXTThis field is used to hold the beginning alpha name or date when the user has elected to run the duplicate patient search using a search criteria of either N (ALPHA RANGE) or R (REGISTRATION DATE). If the user was looking for potential duplicates that were registered between the months of January and February, January would be the SEARCH BEGIN RANGE. Where applicable, the data for the SEARCH BEGIN RANGE prints on the duplicate patient report header.
.13SEARCH END RANGE0;13FREE TEXTThis field holds the ending alpha or date value to search for when the user has selected a duplicate patient search run using the search criteria of N (ALPHA NAME) or R (REGISTRATION DATE). If the user runs a report to find potential duplicate patients for the months of January through February, then February is the SEARCH END RANGE. Where applicable, the data for the field prints on the report header.
