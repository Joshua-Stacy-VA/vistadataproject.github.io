.06units0;6FREE TEXTUNITS represents a measured quantity, such as grams or a number per cubic millimeter or microliter. The field length is defined as 1-10 characters. When entering units, be consistent in the use of upper or lower case for all tests that use the same unit entry.
.07delta check0;7POINTER62.07Delta checks are generally used to compare a current result with a previous result for the same test. The two choices are ABSOLUTE and PERCENT. The ABSOLUTE type of delta check looks for a change in the result based on an exact amount defined in the delta value field. The MTF also defines how many days back the system should check for a delta result. This is defined in the Lab Site file. This method is most frequently used with tests like glucose or potassium. The PERCENT type of delta check looks at the current value and determines if the result has changed greater than the percentage defined in the delta value field. This method is commonly used for tests like WBC or PLT.
.08delta value0;8FREE TEXTThis field contains the numeric value chosen for the type of delta check specified in the `Type of Delta Check' field, either "ABS VALUE" or "PERCENT". Samples which produce results outside of defined delta check ranges are appropriately flagged. The delta check flag is not seen on any reports once the test is certified. It is an internal check for use by lab personnel.
.09default value0;9FREE TEXTIf a particular test has the same answer most of the time (i.e., >80% of the time), entering the common answer in this field will create a "default" answer or prompt. This prompt will allow the tech to enter the common answer by simply hitting the return key when entering, editing or certifying results. If there is no common answer, leave this field blank.
