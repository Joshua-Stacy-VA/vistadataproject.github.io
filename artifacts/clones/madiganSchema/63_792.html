.01MYCOBACTERIUM TB0;1POINTER61.2This field contains the name of the Mycobacterium identified. It points to the Etiology Field file and has a screen so the user can only select Mycobacteria for a result. If a Mycobacterium is not available, then have the lab manager check the Etiology Field file for the entry. If the entry exists in the file, then identifier must be a Mycobacterium. If the entry is not in the file, the lab manager may add it. Make sure the identifier is a Mycobacterium.
.02QUANTITY0;2FREE TEXTThis field is used to quantify the amount of organism isolated. Examples: 1+,2+, 3+ or >100,000 colonies/ml.
.05TB LOINC CODE0;5FREE TEXTThis field is the LOINC Code assigned by the system for this test result. If the system can determine the LOINC code, it is automatically assigned. For Mycobacterium, LOINC codes are captured for each Mycobacterium identified and each antibiotic tested. Data in this field is the actual LOINC code value determined for this test. The data is the same as the .01 field of the LOINC Codes file (file #8199). Note this field is NOT a pointer field. This is to allow test results and LOINC codes to be assigned that may not be in the LOINC codes file. A LOINC code which is not in CHCS may be captured from an off-board system via an HL-7 message when the off-board system is using a different version of the LOINC tables. Data in this field should not be modified by the user and is generally hidden from most data entry mechanisms.
.06TB SNOMED CT0;6FREE TEXTThis field contains the SNOMED CT code generated from the SNOMED encoding process or received through the HL7 interface.
