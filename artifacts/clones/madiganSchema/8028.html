description
This file tracks all the Mass Casualty (MASCAL) events created by a division. Each event name created should indicated the cause of the MASCAL.
Fields
#NameLocationTypeDetailsIndexDescription
.01NAME(+)0;1FREE TEXTBThis field contains the name of a Mass Casualty (MASCAL) event associated with a given division. The name of a given MASCAL event should describe its cause. The individual who initiates the MASCAL should normally designate the name of the associated event. Since there is a possibility of more than one MASCAL event occurring at the same time, the user may have a pick list of events which have previously been defined within their division. In lookup mode, entering the key will bring up the last accessed event.
.02DIVISION(+)0;2POINTER40.8This field tracks the the division where the MASCAL event is exercised. The value of this field is automatically assigned by a trigger on the (.01) NAME field when the MASCAL event is created. It will be set to the division of the user who defines the MASCAL event.
1ACTIVATION DATE/TIME0;3DATE-TIMEThis field defines if a MASCAL EVENT is active and available for user selection when a MASCAL patient is admitted.
2COMPLETION DATE/TIME0;4DATE-TIMEThis field will control if a MASCAL EVENT is considered completed through the MASCAl Parameter process. If this field is populated, the event is closed.
3PREFIX0;5FREE TEXTCThis is another idendifier on the MASCAL EVENT file which will allow the user to differentiate between MASCAL EVENTs easily.
5DESCRIPTION0;6FREE TEXTThis is the free text description of the MASCAL EVENT.
7AUTO ASSIGN MASCAL #0;812This field defines if a MASCAL EVENT has the auto assign MASCAL numbers option enabled. If it is set to YES, the system will fetch the next MASCAL # from the pool for that event. If it is set to NO, the user will have to enter a MASCAL # in the MASCAL admission process.
8LAST MASCAL # ASSIGNED0;9NUMERICThis field stores the last MASCAL # assigned for a given event.
