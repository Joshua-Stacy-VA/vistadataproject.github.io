description
The PROVIDER UNMERGE file records each merge transaction and provides a means of undoing merge transactions for a given provider.

This field indicates that this provider pair was merged/"smooshed" on DEERS. If the provider pair is unmerged on CHCS, the Provider Unmerge functionality will check this field and will then populate all the DEERS information (fields #9 - #14) back into the Provider Merge DEERS file (#8181.5) for any future merges. A value of '1' indicates that "yes" this provider pair has already been merged/"smooshed" on DEERS. A null value indicates that the provider pair has not been merged on DEERS, but has been merged on CHCS.
