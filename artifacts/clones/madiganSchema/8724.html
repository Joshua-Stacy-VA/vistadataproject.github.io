description
This file contains the description of the fields on the label stock that will be used in label printing. It has the field's overall dimensions, contents (whether a picklist or user-definable text field), whether the field will be bar coded, and which bar code format should be used, and the various attributes that the field can have, such as pitch, bold print, underlined. It also specifies truncation requirements and continuation label logistics. Users can defined if and when fields overflow to continuation labels.

 This field determines whether the design definition is used as a primary label, continuation label, or optional label. Continuation labels are used when large data elements on the primary label fill the defined space, and require additional space to print. Continuation labels are not required. Optional labels are used to print data that is not always needed or doesn't always exist. Optional labels allow the printing of labels when there is data to print, while avoiding the printing of data when there is insufficient data to warrant using a label.
.07CONTINUATION DESIGN0;5POINTER8724 This field allows the user to designate a specific label design to be used when the primary label design has more data than will fit in the defined space(s). The continuation design selected must use the same label stock as this primary design and MUST be active. This capability allows the overflow, or 'continuation' label to be user-definable, and have the data elements that caused the overflow condition to be defined on the continuation label in any location the user wishes. This will allow the user to take maximum advantage of the label space provided.
