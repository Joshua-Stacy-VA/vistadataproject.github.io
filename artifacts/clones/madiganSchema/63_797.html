.01date/time result changed0;1DATE-TIMEThis field contains the date and time that a result was changed on DBSS.
.02result0;2FREE TEXTThis field contains the previously reported certified result for a blood bank test before the result was amended on DBSS. Test results are always certifed when transmitted from DBSS to CHCS. When a certified result is changed on DBSS, the new result and its associated required comment are filed as amended results in the result field on CHCS. The previously reported result is stored in this multiple for audit purposes. The result field is free text and contains the result transmitted from DBSS. A certify date and time must be specified for a user outside of the lab to view the result.
.03certify date/time0;3DATE-TIMEThis field indicates the date and time the amended result was certified on DBSS. At this point, the result becomes available to personnel and providers outside of the laboratory.
.04certify person0;4POINTER3This field contains a generic certify person name. Because DBSS does not maintain the CHCS USER file as a Master File, a generic DBSS,USER will be used to populate this field. Refer to the DBSS for the actual name of the user who certified the amended result.
.05initial viewer date/time0;5DATE-TIMEThis field stores the date/time a user first viewed the result.
.06initial viewer0;6POINTER3Ths field stores the internal entry number of the user who first looks at the result via all result reporting options except the Daily Cumulative and Interim reports. If the user is a lab user, the field will not be set.

.08rnr date/time0;8DATE-TIMEThis field stores the date/time that the Health Care Provide or surrogate tossed, printed, saved, or discarded the result in the Review New Results option.

1antibody1;0MULTIPLE63.798This multiple stores the antibodies identified from a positive antibody screen result and the titer results associated with the antibody before it was amended. Each antibody is stored as a separate entry.
