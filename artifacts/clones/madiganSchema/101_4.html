description
This file ques the requests for Order Task creation / discontinuation. The interactive order entry processes call the routine 'ORQUE' which places tasking requests into this file; one entry per Order-request. Orders with priority of 1 (STAT) are tasked off immediately and don't use this process. The Clinical Site parameter, ALLOW ORDER QUE (File#101.1 , field#10.01) , must be enabled for this tasking path to be used. A background job, ORQUE2 (also known as the TASKING DAEMON) periodically checks this file for new entries and processes them. If an entry is successfully processed, its entry in this file is deleted. If a processing error is encountered, it is recorded in the ERROR field and the entry is NOT deleted. All file manipulation is done by the two routines, ORQUE & ORQUE2. FileManager is not used. The zero node fields of LastEntry# and EntryCount are not updated The following nodes are not part of the File Manager file definition for the Orde Que File: |TAB|1. ^ORBQ(101.4,"ERROR") THE ERROR SWITCH [1]= $J of ORQUE2 process [2]= $ZE of last error "ERROR" is set when the Error-State handler encounters too many errors and HALTS ORQUE2. |TAB|2. ^ORBQ(101.4,"RUN") THE RUN SWITCH [1]=$J of ORQUE2 process [2]= Last Processing Run D/T {in FM format} [3]= error count [4]= orders processed "RUN" is set when ORQUE2 is first called to indicate the JOB# of the process running ORQUE2. Can be used to trap and eliminated multiple simultaneous running ORQUE2s. |TAB|3. ^ORBQ(101.4,"HALT") THE HALT SWITCH [1]= "" "HALT" can b set to stop the Tasking Daemon.
