This is a control file that is used to store site configurable parameters for the operation of the interface.

.01name(+)0;1POINTER4.3BThis is the name of the interface site where the GIS is running. It is expressed in the format of an Internet Domain Name, e.g., FWVA.SAIC.COM
.02default max number of attempts(+)0;2NUMERICThis indicates the maximum number of times that transactions will be requeued. This is a system default. The system will first check the transaction type, then the destination to determine the maximum number of retries. If no value is found then the value of this field will be used. If no value is present in this field the default is zero (0).
.03default retry rate(+)0;3FREE TEXTThis is the default amount of time between attempts to process entries in the UIF. Only PENDING entries will be retried. Enter a number of hours (nnH), number of minutes (nnM), or a number of days (nnD).
.06priority for background tasks0;6NUMERICThis field is used to indicate the priority at which spawned background tasks should be run. A value of 0 or null will use the default priority. Otherwise, 1 is lowest priority, 10 is highest and 5 is normal.
.07general purpose output servers(+)0;7NUMERICThis field determines the number of processes which will be created by the Output Controller fail-safe process. Entering a value greater than one will cause the Output Controller to operate in multi-thread mode, in which additional jobs will be created to perform the tasks. This is the number of Output Controller server processes and must be greater than zero.
.08message id prefix(+)0;8FREE TEXTThis is the standard prefix which will be used to create Message ID's. For example, if the prefix is HOST the first message created will have an ID of HOST1, the second will have HOST2, etc.
.1formatter servers0;10NUMERICThis is the number of Format Controller Server processes that will be created by the Format Controller fail-safe process.
.11uif/error retention days(+)0;11NUMERICThis is the number of days old a transaction (in error, complete, or negative acknowledged state) or an error must be in order for it to be purged by the GIS autopurge program.
.12variable storage location0;12SET OF CODES0:LOCAL, CAN MOVE TO GLOBAL
1:LOCAL
2:GLOBAL
This field describes where data processed by an interface script is to be held. If LOCAL, CAN MOVE TO GLOBAL the data will be kept in a local array until the local storage gets too low then it will be moved to a global. The overhead of this method is that the checking needs to be done periodically. If LOCAL is chosen, data will be kept in a local array and no checking will be done for remaining storage - only use this method when you are sure the amount of data will not exceed local storage. If GLOBAL is chosen, data will be kept in a global and no checking will be done for remaining storage - use this method when you know that almost all transactions will overflow the local storage.
.13minutes to be current(+)0;13NUMERICThis is maximum number of minutes old any pending or failed messages may be for the Interface System to be deemed "current". If there are any transactions with a status other than complete outside this number of minutes, then the interface system will report itself as "not current".

2.07purge per task2;7NUMERICThe value in this field will be multiplied by one thousand. The resulting value determines the maximum number of Universal Interface entries that will be examined by the interface purge process. If the number entered is less than the number of records requiring review for purge eligibility, additional purge tasks will follow until completion or shutdown. Any positive number entered is interpreted in 1K units. If the field is left blank, a single process will be used for the entire purge.
2.08shutdown purge time2;8FREE TEXTThe value of this field indicates when the purge task must stop processing. The value 0000 will not be accepted because it is ambiguous.
2.09retask delay seconds2;9NUMERICThis value defines the number of seconds to delay the tasking of subsequent purge jobs. Tasking of the initial day's purge job is unaffected by the value entered, and it will start on the time set for the Acceptable Purge Time field. If the value of the Purge Per Task field is null or greater than the number of records for purge eligibility review, one tasked job will be used for continuous record purging.

4.01max failed real deers connects4;1NUMERIC This field specifies a threshold for the Maximum # of Failed Interactive Connection attempts for the HIPAA X12 Interface. When this threshold is exceeded, users are notified to take action.

At the end of each 30-minute interval the count of failed batch attempts will be reset, and the tracking of failures will begin again.
