This field defines the physical type of LSI that is being used. There are several types of LSI that can be used, some utilize an actual hardware device that concentrates the data from multiple instruments onto one communications link to CHCS. Others are simply logical representations of a direct communications link between CHCS and a single instrument. Select the LSI type that is appropriate for this entry: LAASTM is used for directly connected instruments that utilize the ASTM (American Society for Testing and Materials) Standard #1381-91. The title of this specification is: Specification for Low-Level Protocol to Transfer Messages Between Clinical Laboratory Instruments and Computer Systems. LACOM is used when the LSI is a PDP-11/53 minicomputer. This LSI type will only be used to connect uni-directional instruments to CHCS. These instruments simply transmit their data without any need for a response from the receiving system. LAGIS is used when the LSI is a DATA INNOVATIONS Instrument Manager LSI. This LSI uses a personal computer to connect the instruments to CHCS. It also performs translation of the custom instrument data, converting the data into a minimal HL-7 data format for transmission to CHCS. This LSI utilizes the CHCS GIS (Generic Interface System) for communications between CHCS and the LSI. All instrument interface issues are handled the DII IM system. LALONE is used when interfacing a bi-directional instrument directly with the CHCS system. This type of LSI is used for most directly connected instrument interfaces. However, there are other types that can and/or should be used, depending on the needs of the instrument protocol. Please check the deployment guide or Instrument Interface Parameters file for the instrument being interfaced to determine if this is the type of LSI that should be used. LAPOLL is a variation of LALONE. It is used for directly connected instruments that required CHCS to occasionally poll the instrument for data to be transmitted. LAPOLL will transmit to the instrument an instrument-specific poll message requesting any available data be transmitted to CHCS. This poll message is transmitted at instrument-specific intervals. Please check the deployment guide or Instrument Interface Parameters file for the instrument being interfaced to determine if this is the type of LSI that should be used. LASTAR is another variation of LALONE that is also used for directly connected instruments. LASTAR is used when the format of the data from the instrument contains large amounts of text without message termination characters or when the message termination characters must be retained for effective processing of the data. LALONE will not retain the message termination characters, but LASTAR will. LASTAR can also accept a virtually unlimited amount of data without a message terminator character. Please check the deployment guide or the Instrument Interface Parameters file for the instrument being interfaced to determine if this is the type of LSI that should be used. LASYS is used when the LSI is a GDC 2100 microcomputer. This LSI type is the first generation of LSI hardware devices, and is only used to connect uni-directional instruments to CHCS. These instruments simply transmit their data without any need for a response from the receiving system.
