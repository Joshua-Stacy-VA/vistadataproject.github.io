description
This file contains an entry for each HEALTH CARE PROVIDER holding file created for RCMAS-OSE (Retrospective Case Mix Analysis System - Open System Environment). Each entry documents one occurrence of data having been captured from the PROVIDER File into a VMS flat file in preparation for transfer via Kermit to the RCMAS-OSE environment.

.01date(+)0;1FREE TEXTThe DATE field is the range of dates in ^DIC(6,"AF" that are used to build the Health Care Provider file (e.g., May 1,1993@1015-May 31,1993@1400). The DATE field is stored in FILEMAN format. In the example above, the DATE field would be 2930501.1015-2930531.1400.
.02provider file type(+)0;2SET OF CODESM:MONTHLY HCP
I:INTERIM HCP
The PROVIDER FILE TYPE field is a set of codes representing the Health Care Provider file type (M=Monthly, I=Interim). The Monthly Health Care Provider file is created when the user runs the monthly SIDR tape option to produce the monthly SIDR tape. The Interim Health Care Provider file is created when the user selects to build an Interim SIDR file on-demand; the Health Care Provider file is generated at the same time as the SIDR file.
.03file status(+)0;3SET OF CODES0:FILE NOT READY
1:FILE READY
The file status indicates whether there is data ready for transmission (1=ready to transmit) or that data is being written to the file to be transmitted (0=not ready to transmit).
.04number of provider records0;4NUMERICThe NUMBER OF PROVIDER RECORDS field is the number of physical records in the Health Care Provider file. Each provider counts as one record.
.06date/time last transmitted0;6DATE-TIMEThe DATE/TIME LAST TRANSMITTED field is the date/time that the Health Care Provider file was last transmitted to RCMAS-OSE. The Health Care Provider file is transmitted to RCMAS-OSE on demand via the RCMAS option.
.07user who built file0;7POINTER3The USER WHO BUILT FILE is the internal entry number (DUZ) of the user who initiated the creation of the file. For Monthly Health Care Provider files, the USER WHO BUILT FILE is the user who generates the monthly SIDR tape. For Interim Health Care Provider files, the USER WHO BUILT FILE is the user who requestse to build the Interim files via the RCMAS option.
.08date/time file built0;8DATE-TIMEThe date/time that a specific CHCS health care provider holding file was built in preparation for transfer to the RCMAS-OSE environment.

The FILE DELETE FLAG is a set of codes indicating whether the file containing the actual Health Care Provider data has been deleted from the system (1=DELETED).
.1division(+)0;10POINTER40.8The DIVISION field is the division that the HCP file was created for (pointer to file #40.8). The DIVISION field will be automatically set to the user's division when the HCP file is created.
