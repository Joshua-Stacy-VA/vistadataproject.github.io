description
The QUALITY CONTROL file is a dictionary for Laboratory controls used in the JCAH and CAP required quality control program. Entries may be added, but, once added NEVER deleted since verfied results may be associated with the control name. If you want to deactivate the control name, simply enter "Yes" to the STOP DATA ENTRY field.

.01name(+)0;1POINTER2BThis field contains the name of the control. The name must be unique for each control. When the control outdates frequently, include the expiration date in the test name. Lab personnel can then recognize immediately if the wrong control is accessioned on a work document.
.02identifier0;2FREE TEXTThis free text field allows you to enter the control lot number, code or other manufacture or lab identification.
.03expiration date0;3DATE-TIMEThe expiration date of the control. This is used as a reference date only. The system continues to accept data after the expiration date, unless otherwise specified in STOP DATA ENTRY field. This date must be continually updated if the lab chooses to extend the expiration date. The Non-Historical Purge routine will purge QC's out of the file IF the Stop Data Entry field is set to 'Yes' AND 13 months have passed since the expiration date. If the expiration date is not kept current, then results may be purged before they should be.
.04stop data entry0;412Set this field to 1 (YES) if you want to stop data entry for this control. This control can't then put to the work document or accessioned. This field is used by the software to determine if QC's and their results are to be purged from the system. The Non-Historical Purge routine purges the QC's IF the Stop Data Entry field is set to 'Yes' AND 13 months have passed since the expiration date. If this field is not correctly updated, then data that should be purged from the system won't be.
