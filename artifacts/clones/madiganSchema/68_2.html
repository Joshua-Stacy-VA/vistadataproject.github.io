description
The Work Document file contains the specific site information related to each work document. This file must first be defined before building and using the work documents. Each work document obtains the ordered test from its associated accession area. An ordered test for a specific patient can only be on one work document at a time. When a work document is "unloaded" the uncertified tests return to the associated accession area pool.

Enter one of the formats for the work document: Vertical: The tests are arranged in columns below the patient and accession information. Load List: The tests ordered are listed on a single line, separated by commas, following the patient and accession information. If the tests are resulted and the work document is printed the format returns to the vertical format so the results can be reviewed. The load list can also be set up to expand the panels when it is printed. When a panel prints all the tests print on the panel are displayed separated by commas. Horizontal: The patient and accession information are minimal and print to the left side of the work document. The tests print across the top of the work document. If the accession has the test ordered a "___" prints under the test header. If the test is resulted the result prints. This format will only handle up to 10 tests, and the tests would need to have names and results that do not take up too many spaces. Template: The lab manager uses the Inquire to the Files option of the Ad Hoc Menu to create a custom template for a work document. The patient and accession information is followed by the contents of the custom template. If this format is chosen, one entry prints per page. 3/4 Columnar: The patient and accession information is followed by the panel tests listed vertically in 3 or 4 columns. The results also print on the work documents in the columns. 5 Spaces or Lines: The patient and accession information is followed by either 5 double spaced lines or 10 spaces. This provides a space for a user to write notes about the test for reference when the final result is entered.

.09SORT ORDER0;9SET OF CODESA:ACCESSION
P:PRIORITY
Use this field to designate if the work document should sort by accession number or by priority. If the sort order is by priority, the accessions will appear on the work document with STATs and other high priorities printing first. Within each priority, the accessions will sort by the date of the accession. For example, routine priorities will have the oldest accessions appearing at the top of the routine portion of the list. If the sort order is by accession, the entries on the work document will print in numerical order of the number part of the accession. Neither priority nor the date of accession will be used in the sort critieria.

30BUILDING IN PROGRESS3;112While the work document is being built, the program sets this flag to YES. At the completion of building, this flag is returned to NO. The purpose is to keep multiple users from competing for the same work document at the same time. If the flag is left set, enter "NO" to continue building load lists. When the field is set at "YES" the user will see the prompt "WAITING FOR ANOTHER BUILD TO FINISH" when attempting to build the list. If no other user is attempting to build the same list, check to see if this field needs to be reset.
50TEMPLATE10;0MULTIPLE68.23This is a multiple entry field. Different combinations of tests may be defined to include on the particular work document. Each work document must have at least one template defined, but may have other templates. When more than one template exists, the users will be prompted with "Build ALL templates?" when building the document. The user may choose to build all existing templates onto the same document or can specify one template. If only one template is defined, the user will not see this prompt. The simplest example of a template might be for a single test. The work document name is "Lithium", the template defined is named "Lithium", and the only test included on the template is "Lithium". Certain circumstances may require two templates. There may be a template called "Standard" for the regular day shift operations and a "PM/MN" template for the off-peak hours.
