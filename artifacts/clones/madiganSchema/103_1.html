2.01activity2;1SET OF CODES0:INACTIVE
1:ACTIVE
2:PENDING
This is the highest level indicator of order status. An active order is one which is currently being processed. If pending, the order has been temporarily suspended pending some action which must take place, such as a nursing signature or an appointment made. If inactive, this particular order has either been resulted, expired or cancelled/discon- tinued.
2.02active status2;2SET OF CODES0:FULLY ACTIVE
1:SIGNATURE ABSENT
This field further defines the status of an ACTIVE order. A FULLY ACTIVE order means that there are no conditions placed on the order's activity. SIGNATURE ABSENT means that the HCP and/or COUNTERSIGNING HCP have not yet signed the order, although the order is active because a nurse has signed it. This flag is used to set the 's' or 'c' notation on the display screen.
2.03inactive status2;3SET OF CODES0:RESULTED
1:DISCONTINUED
2:TRANSFERRED
3:SIGNATURE MISSING
4:RESULTED-NO SIG
This flag further defines the status of an inactive order. If RESULTED, the order has had all tasks resulted. EXPLICITLY DISCONTINUED means that an action was taken by a provider to stop this order. TRANSFERRED is the inactive status created when a patient is transferred within the MTF. At that point, all orders are 'discontinued' but the provider has an opportunity to later mark which of these orders he would like to 'reactivate'. This reactivation actually results in new orders being generated. SIGNATURE MISSING means that an order has been discontinued but still needs a signature or countersignature. It will continue to show up on the display screen until signed. If a provider cancels an order requiring a signature, and he is qualified to sign that order, the signing takes place automatically and this status is avoided.
2.04pending signature2;4SET OF CODES1:PENDING SIGNATURE
This pending flag indicates that an order has not been activated because either a nurse or HCP signature has not yet been obtained.
2.05pending appointment2;5SET OF CODES1:PENDING APPOINTMENT
This pending flag indicates that an order is suspended while awaiting completion of the scheduling process. Because one order may generate several tasks requiring scheduling, this flag is set when NONE of the remaining tasks have been scheduled - in other words, when there is a potential hold up of the process as a result of schedules not being completed. Once the first task is scheduled, the order is active unless the first task is completed and the next task is not yet scheduled.
2.06pending activation2;6SET OF CODES1:PENDING ACTIVATION
This pending flag indicates when an order has been placed on hold. This status changes either when the user activates the order or when the ACTIVATION DATE arrives and the order is taken out of suspense automatically. In this instance, the order is then placed in PENDING REVIEW (see next field).
2.07pending review2;7SET OF CODES1:PENDING REVIEW
This pending flag is set when an order on HOLD (PENDING ACTIVATION) reaches the activation date. Rather than automatically activate the order, this status alerts the user that the order is due to be activated after he/she has reviewed it for its appropriateness at this time.
2.08activation date2;8DATE-TIMEWhen an order is put in suspense (as for preadmission orders), the user is given the option of indicating an automatic reactivation date. The suspended order is changed from Pending Activation to Pending Review when this date has been reached.
2.09priority(+)2;9POINTER102.3This field indicates the urgency with which an order should be carried out. For lab orders, it indicates the priority for lab collection (how quickly should a specimen be obtained). The two standard priorities are STAT and ROUTINE. This field points to the Order Priority file in which the MTF can place additional priority levels if it wishes. If the order has STAT priority, the system will be monitering the time to acknowledge the order and then the time to result the order. Times exceeding MTF defined intervals will be tracked for QA.
2.1warning action required2;10SET OF CODES1:DISPOSITION NEEDED
This pending flag will be set when an order which requires a warning to be overridden in order to become active, has not yet been over- ridden. A nurse may override the order either under authority of an HCP or as an emergency override. The HCP has the ability to either override the order or kill it. In all cases, the disposition of a warning will be stored for QA.
