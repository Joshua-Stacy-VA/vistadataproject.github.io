.01error0;1SET OF CODES99:no lock
1-12:see field description
This field contains the error codes which represent a problem with data being filed in the PSRX global. A 99 implies a locking problem and this code is only returned from routine PSOSTUF. All other codes (1-12) are returned from routine PSOSRX which in turn are defined by routine ORNEW3. The codes from ORNEW3 are in the variable, ORN("BAD"). These codes and definitions are stored in ERRMSG+code^PSOSRX. The values and definitions for the ORNEW3 codes are as follows: 1 = missing OE data 2 = no HCP class 3 = no order privileges 4 = HCP authorization error 5 = order type not in file 6 = no order processing array 7 = fileman lookup failure 8 = signature class problem 9 = Order Type file invalid 10 = default order set template not defined 11 = no order# assigned 12 = not an Rx order type
.02error description0;2FREE TEXTThis field describes the type of error (represented by the error code) encountered in the filing of prescription data in the PSRX global. A time stamp of when the error was filed will precede the text.
