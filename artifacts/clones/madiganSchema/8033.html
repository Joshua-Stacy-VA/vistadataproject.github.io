description
This file contains SIT/OHI system parameters. These parameters may be accessed by a programmer only.
Fields
#NameLocationTypeDetailsIndexDescription
.01sit message wait time(+)0;1NUMERICBDefines the length of time which will be allowed for the interface to access a lock in an attempt to process an inbound SIT message. When the attempt to access a lock exceeds this wait time, an error will be logged.
.02ohi message wait time(+)0;2NUMERICDefines the length of time which will be allowed for the interface to access a lock in an attempt to inprocess a OHI message. When the attempt to access a lock exceeds this wait time, an error will be logged.
.03error log purge days(+)0;3NUMERICThe number of days that an error may remain in the SIT/OHI ERROR LOG File #8031. After this time, the error will be purged. The default is 4 weeks.
.04collection file purge days(+)0;4NUMERICThe number of days that message information may remain in the SIT/OHI COLLECTION File #8032. After this time, the message information will be purged. The default is 4 weeks.
.05lock number of tries(+)0;5NUMERICNumber of times that the interface will attempt to access a lock. After the specified number of attempts, an error will be logged.
.06sit/ohi switch flag0;6SET OF CODES1:ON
0:OFF
The SIT/OHI SWITCH FLAG is used to implement the new TNEX SIT/OHI software changes. When the software is installed, the SIT/OHI SWITCH FLAG is set to "O", meaning that the previous version of the software remains in effect. When the SIT/OHI SWITCH FLAG is changed to "1", the conversions are run, the new menus display, and the new functionality is in effect.
.07transaction system name0;7FREE TEXTThe Transaction System Name is assigned by DMDC to each external system which interacts with DEERS via the TNEX SIT/OHI interface. The Transaction System Name in this parameter file indicates how the local system is identified in the CHCS/DEERS SIT/OHI interface.
.08last subscription inquiry date0;8DATE-TIMEDate that CHCS last sent a SIT Subscription Inquiry to DEERS and a response was successfully received.
.09transaction system id0;9FREE TEXTThis field contains an HCDP System Identifier assigned by DEERS. The value in this field reflects the identification of the system from which SIT/OHI record originated, or the system from which the carrier/policy-level updates were last transmitted to DEERS.
.1site id0;10FREE TEXTSite ID that DMDC assigns to each site.
.11esi object port number0;11NUMERICThe port number that Esi Object Server will be started on.
