.04priority0;4SET OF CODES0:LOW
1:HIGH
When the priority is set to "HIGH", BD/URS transmissions will be processed before DEERS transmissions. If only one server is available, no DEERS requests will be processed during the BD/URS transmission period (DURATION). When the priority is set to "LOW", DEERS transactions will be processed before BD/URS transmissions.
.05start time0;5FREE TEXTThis is the time that the BD/UR's job will be scheduled to run.
.06bd/urs site id0;6FREE TEXTNumber provided by the government DEERS Project Office to the MTF. This number is sent from CHCS to DEERS with each BD/URS transaction. It is a six digit decimal number. If a HEX number is supplied, convert the number into decimal. Usually, the BD/URS site ID and DEERS site ID will be the same. If no site ID is provided in this field, BD/URS will automatically use the site ID provided in the DEERS site parameter file.
.07wait time0;7NUMERICThe external BD/URS process requires time between transactions. This time may vary from site to site depending on how the link is shared with other systems. This parameter allows a site to adjust the wait time if needed. The time is in seconds.
.08maximum servers0;8NUMERICThis is the maximum number of servers for BD/URS processes.
