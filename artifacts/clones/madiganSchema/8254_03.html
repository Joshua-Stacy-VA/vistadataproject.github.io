description
PS OBSOLETE NDCS This file contains National Drug Codes (NDCs) that have become obsolete and a replacement NDC is known. This is used to convert site files that are using the deleted NDC to the replacement NDC. It is also used to redirect an allergy selection entry (8254.01) to a new Allergy Definition when applicable. Its IEN is the deleted NDC number with leading zero and all dashes removed.

.02gcnseqno(+)0;2POINTER8251This field contains the GCN sequence number. The GCN sequence number is a random number representing the generic formulation. It is specific to the generic ingredient combination, route of administration, dosage form, and drug strength. In addition, the GCN*SEQNO is the same across manufacturers and package sizes. However, the GCN*SEQNO is an absolutely unique number; it cannot be duplicated. The number by itself has no significance. No digit or group of digits within the GCN*SEQNO has any more significance than the rest of the number. However, it is very useful when used for on-line computer applications, such as generic substitution. The GCN*SEQNO can be used to group pharmaceutically equivalent products together. (Note: Products having the same GCN*SEQNO are not necessarily bioequivalent.)
.03obsdte0;3DATE-TIMEThis field contains the date that this National Drug Code (NDC) has been identified by the manufacturer as an obsolete (discontinued) NDC number. When an NDC has been discontinued, First DataBank assigns an obsolete date to the NDC record. First DataBank will stop updating this NDC after a cutoff obsolete period agreed upon by the customer. This cutoff obsolete period is typically 3 years, so that if, for example, an NDC contained an obsolete value of 890101, January 1, 1989, then on any update or full file tape created on or after January 1, 1992 (3 years after 890101), the NDC would no longer be sent. Customers need to know that the NDC is no longer being supported on their update tapes, so that the customer system can either anticipate when the NDC will no longer be sent, based on the cutoff date the customer provides to First DataBank, or by receiving a full file replacement formulary on some regularly scheduled basis, or through ad hoc requests. This field can be used in conjunction with the Previous National Drug Code (PNDC) and the Replacement National Drug Code (REPNDC).

The field has a non-blank value only on those products that have been identified by the manufacturer as being replaced by the new NDC number (REPNDC). All replacement NDC's are structured in the eleven-digit format. This field is usually used in conjunction with the Obsolete Date (OBSDTE) and the Previous National Drug Code (PNDC).
