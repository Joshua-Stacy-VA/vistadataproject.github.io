EX: 1234 reversed is 34^12) test = process test, pointer to #60 TYPE = IN or OUT patient flag I=in, O=out Special technical information regarding all fields that contain the word LOINC in their field name. The acronym LOINC is for (L)ogical (O)bservation (I)dentifiers, (N)ames, and (C)odes. The various LOINC Code fields are the actual LOINC Code assigned by the system for a test result. If the system can determine the LOINC code, it is automatically assigned. For Clinical Chemistry tests, LOINC codes are derived by the system based on the component name of the test, the site/specimen for the accession, and lab method used during result entry. The Lab Method contains the necessary references to determine the appropriate LOINC Time Aspect, LOINC Property, LOINC System/Sample (which is further linked via the Topography Field file), LOINC Scale, and optionally LOINC Method. For Bacteriology accessions, a LOINC Code is captured for the GRAM STAIN field (if this field is populated during result entry), for each microorganism identified, and for each antibiotic tested. For Autopsy, Bone Marrow, Cytology GYN, Cytology Non-GYN, and Surgical Pathology accessions, a single LOINC code is assigned for each accession. This LOINC code applies to all result entry fields for that accession. For Mycology accessions, a LOINC code is captured for each Fungus/Yeast identified. For Parasitology, a LOINC code is captured for each parasite identified. For Mycobacterium, LOINC codes are captured for each Mycobacterium identified and each antibiotic tested. For Virology, a LOINC code is captured for each virus identified. Tests in the BB (Blood Bank) subscript currently are not assigned LOINC codes. CHCS is not considered the source of blood bank test results, the DBSS (Defense Blood Standard System) is the source. LOINC Codes should be derived on the system that is the source of the test results. At this time (March 1999), the DBSS system and the HL-7 interface between CHCS and DBSS does not support the exchange of LOINC codes for Blood Bank test results, antibody identifications, or blood product management. Data in these fields are actual LOINC code values determined by the system for a particular test result. The data have the same values as the .01 field of the LOINC Codes file (file #8199). Note these fields are NOT pointer fields. This is to allow test results and LOINC codes to be assigned that may not be in the LOINC codes file. A LOINC code which is not in CHCS may be captured from an off-board system via an HL-7 message when the off-board system is using a different version of the LOINC tables. Data in these fields should not be modified by the user and are generally hidden from most data entry mechanisms. Only authorized support personnel should change data in the LOINC code fields.
