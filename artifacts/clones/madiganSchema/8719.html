description
This file contains information about shipping batches and the accessions on them. The data is created when batches are generated both on the submitting laboratory and the performing laboratory. This data is purged by the nightly purge utility when the specified purge criteria are met. This file is designed and developed for the use of the Interoperability project. This file is maintained by the Interoperability software. DO NOT EDIT THE FILE ENTRIES. This file also contains a non-FileMan field on the ORDERED TEST multiple field (sub-file 8719.05). This non-FileMan field contains the INV array formatted by the GIS when the order/accession message is received. It is used when the specimen is acknowledged and is deleted when the specimen is accessioned at the performing laboratory.
