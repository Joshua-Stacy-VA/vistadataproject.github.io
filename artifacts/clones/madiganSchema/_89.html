name
COMMAND SCREEN
number
.89
location
^DWA(.89,
description
This file contain all the command screen definitions that developers set-up.. The 'Command Screen Processor' uses the information defined in these files. The 'commands' that a user can invoked during a command-screen seesion are defined in the COMMAND ACTION file.

.01name(+)0;1FREE TEXTBEnter the name of the command screen in the same format as you would name a template. The namespacing convention should be followed.
.02type0;2SET OF CODES1:ACTION
2:LIST PROCESSOR
3:BAR
There three types of screen: 1 ACTION This type of screen will display a list of items and have a menu of actions associated with it. The cursor will be positioned at the prompt for the action. 2 LIST PROCESSOR CALL ONLY This type of screen will simply set up the variables from the Command Screen file, call the List Processor and quit. 3 BAR This type will associate an Action Bar with the List Processor. The user will be able to process actions from the action bar and be able to control the list simultaneously.
